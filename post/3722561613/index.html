基础语法 - MySQL

1. 通用语法

SQL 语句可以分为以下四类：

分类 说明
DDL 数据定义语言，用来定义数据库对象（数据库、表、字段）
DML 数据操作语言，用来对数据库表中的数据进行增删改查
DQL 数据查询语言，用来查询数据库中表的记录
DCL 数据控制语言，用来创建数据库用户、控制数据库的访问权限

SQL 的通用语法如下：

  • SQL 语句可以单行或多行书写，以分号结尾。
  • SQL 语句可以使用空格/缩进来增强语句的可读性。
  • 注释：
    • 单行注释：--注释内容或#注释内容
    • 多行注释：/*注释内容*/
  • MySQL 数据库的 SQL 语句不区分大小写，关键字建议使用大写

2. 数据类型

2.1 数值类型

类型 大小 有符号(signed)范围 无符号(unsigned)范围 描述
tinyint 1 byte (-27, 27-1) (0, 255) 小整数值
smallint 2 bytes (-215, 215-1) (0, 216-1) 大整数值
mediumint 3 bytes (-223, 223-1) (0, 224-1) 大整数值
int 或 integer 4 bytes (-231, 231-1) (0, 232-1) 大整数值
bigint 8 bytes (-263, 263-1) (0, 264-1) 极大整数值
float 4 bytes (-3.402823466E+38, 3.402823466E+38) （IEEE 754） 0 和 (-1.175494351E-38, 3.402823466E+38) 单精度浮点数值
double 8 bytes (-1.7976931348623157E+308, 1.7976931348623157E+308) （IEEE 754） 0 和 (2.2250738585072014E-308, 1.7976931348623157E+308) 双精度浮点值
decimal 依赖于M(精度)和D(标度)的值 依赖于M(精度)和D(标度)的值 小数值(精确定点数)

有符号类型在书写时可默认省略 signed，无符号类型在书写时采用 类型 unsigned。

浮点数通过括号内的数字限定总数字位数和小数位数：double(总位数，小数位数)。

2.2 字符串类型

类型 大小 描述
char 0-255 bytes 定长字符串
varchar 0-65535 bytes 变长字符串
tinyblob 0-255 bytes 不超过 255 个字符的二进制字符串
tinytext 0-255 bytes 短文本字符串
blob 0-65535 bytes 二进制形式的长文本数据
text 0-65535 bytes 长文本数据
mediumblob 0-16777215 bytes 二进制形式的中等长度文本数据
mediumtext 0-16777215 bytes 中等长度文本数据
longblob 0-4294967295 bytes 二进制形式的极大文本数据
longtext 0-4294967295 bytes 极大文本数据

定长字符串通过括号内的数字限定字符数：char(字符数)。固定占用指定大小的空间，性能优于变长字符串。

变长字符串通过括号内的数字限定字符数：varchar(字符数)。占用的空间由实际存储的字符数决定。

4.4 流程函数

函数

流程函数可以在 SQL 语句中实现条件筛选，从而提高语句的效率

  • 条件函数 if：如果 value 为 true，则返回 t，否则返回 f

    if(value, t, f)

  • 判断为空函数 ifnull：如果 value1 不为 null，返回 value1，否则返回 value2

    ifnull(value1, value2)

  • 分支选择函数 case when

注意

分支选择函数 case when 当一个条件满足，就返回对应的结果，不会继续下一条件判断。

5. 正则表达式

语法

  • 指定正则表达式的字符匹配模式

    字段 regexp '正则表达式'

格式

MySQL 中的正则表达式与 Java 中的正则表达式基本一致，常用匹配方式如下：

模式 描述
^ 匹配输入字符串的开始位置
$ 匹配输入字符串的结束位置
. 匹配除"\n"之外的任何单个字符
[] 字符集合，匹配所包含的任意一个字符
[^] 负值字符集合，匹配未包含的任意字符
| 连接多个匹配条件表示逻辑或
* 其前导字符必须在目标对象中出现零次或连续多次
+ 其前导字符必须在目标对象中连续出现一次或多次
{n} 其前导字符正好出现 n 次
{n,m} 其前导字符至少出现 n 次，但不超过 m 次
**注意**
  • regexp 和 like 的区别为：regexp 只需要匹配上列值中部分字符串，但 like 需要匹配上整个字符串。
  • Java 和 MySQL 的正则表达式的区别：Java 中 matches() 需要匹配上整个字符串，regexp 只需要匹配上列中部分字符串。
  • MySQL 中 \ 为转义符，正则表达式中 \ 也为转义符。
  • 字符串中要输出特殊符号本身必须用 \符号，正则表达式中要输出特殊符号本身必须用 \\符号。
  • 字符串中输出 \ 符号本身必须使用 \\，正则表示中输出 \ 符号本身必须使用 \\\\

6. 性能分析

SQL 执行频率

  • 查看服务器状态信息

    show [session|global] status;

  • 查看当前数据库增删改查语句的访问频次

    show global status like 'Com_______'; （七个下划线）

慢查询日志

慢查询日志（文件：主机名-slow.log）记录了所有执行时间超过指定参数（long_query_time，单位：秒，默认10秒）的所有 SQL 语句的日志。

MySQL 的慢查询日志默认没有开启，需要在 MySQL 的配置文件（/MySQL Server 8.0/my.ini）中配置如下信息：

profile 详情

show profiles 能够在做 SQL 优化时帮助我们了解时间都耗费到哪里去了。

  • 通过 have_profiling 参数，能够看到当前 MySQL 是否支持 profile 操作

    select @@have_profiling;

  • 默认 profiling 是关闭的，可以通过 set 语句在 session/global 级别开启

    set profiling = 1;

在开启 profiling 之后可以通过如下语句进行查询：

  • 查看每一条 SQL 的耗时基本情况

    show profiles;

  • 查看指定 query id 的 SQL 语句各个阶段的耗时情况

    show profile for query 指定id;

  • 查看指定 query_id 的 SQL 语句 CPU 的使用情况

    show profile cpu for query 指定id;

explain 执行计划

直接在 select 语句前加上关键字 explain 或者 desc ，可以获取 MySQL 如何执行 select 语句的信息。

获得的执行计划各字段含义：

  • id：select 查询的序列号，表示查询中执行 select 子句或者是操作表的顺序（id 相同，顺序从上到下；id 不同，值越大，越先执行）
  • select_type：表示 select 的类型，常见的取值有 SIMPLE（简单表，即不使用表连接或者子查询）、PRIMARY（主查询，即外层的查询）、UNION（union 中的第二个或者后面的查询语句）、SUBQUERY（select/where 之后包含了子查询）等
  • type：表示连接类型，性能由好到差的连接类型为 NULL、system、const、eq_ref、ref、rang、index、all
  • possible_key：可能用到的索引
  • key：实际使用的索引，如果为 NULL，则表示没有使用索引
  • key_len：表示索引中使用的字节数，该值为索引字段最大可能长度，并非实际使用长度，在不损失精确性的前提下长度越短越好
  • rows：MySQL 认为必须要执行查询的行数，在 InnoDB 引擎的表中是一个估计值
  • filtered：表示返回结果的行数占需读取行数的百分比，该值越大越好
