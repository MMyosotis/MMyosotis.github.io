1. 创建多线程

线程是一个程序内部的一条执行路径。程序中如果只有一条执行路径，那么这个程序就是单线程的程序，main 方法的执行就是一条单独的执行路径。

多线程是指从软硬件上实现多条执行流程的技术。Java 是通过 java.lang 包下的 Thread 类来代表线程的，Thread 类应该提供了实现多线程的方式。

1.1 继承 Thread 类

通过继承 java.lang 包下的 Thread 类可以实现多线程，步骤如下：

  • 定义一个线程类继承 Thread 类，重写 run() 方法，定义线程内部干嘛。
  • 创建线程类的线程对象。
  • 调用线程对象的 start() 方法启动线程（启动后还是执行 run 方法的）。

该方法的注意事项如下：

  • 线程类已经继承 Thread，无法继承其他类，不利于扩展。
  • run 方法为 void，不能定义有返回值的操作。
  • 直接调用 run 方法会当成普通方法执行，此时相当于还是单线程执行。只有调用 start 方法才是启动一个新的线程执行。
  • 若把主线程放在子线程 start 方法之前，这样主线程一直是先跑完的，相当于是一个单线程的效果了。

1.2 实现 Runnable 接口

在 java.lang 包下的 Runnable 类提供了实现多线程的方式，步骤如下：

  • 定义一个线程任务类实现 Runnable 接口，重写 run() 方法。

  • 创建线程任务对象。

  • 使用构造方法创建 Thread 对象，把任务对象交给 Thread 处理。

    构造方法 方法说明
    public Thread(String name) 可以为当前线程指定名称
    public Thread(Runnable target) 封装 Runnable 对象成为线程对象
    public Thread(Runnable target, String name) 封装 Runnable 对象成为线程对象，并指定线程名称

  • 调用 Thread 线程对象的 start() 方法启动线程。

该方法注意事项如下：

  • 线程任务类只是实现接口，可以继续继承类和实现接口，扩展性强。
  • run 方法为 void，不能定义有返回值的操作。

1.3 实现 Callable 接口

JDK5 新增了 Callable 接口和 FutureTask 类（从 Runnable 逐层继承下来的）来实现多线程，步骤如下：

  • 定义类实现 Callable<V> 接口并指定泛型（设定为返回值类型），重写 call 方法，封装要做的事情。

  • 使用 FutureTask 的构造方法，把 Callable 对象封装成线程任务对象。

    构造方法 方法说明
    public FutureTask<>(Callable call) 把 Callable 对象封装成 FutureTask 对象

  • 使用 Thread 的构造方法，把线程任务对象交给 Thread 处理，包装成线程对象。

  • 调用 Thread 的 start 方法启动线程，执行任务。

  • 线程执行完毕后，通过 FutureTask 的 get 方法去获取任务执行的结果。

    方法名称 方法说明
    public V get() throws Exception 获取线程执行 call 方法返回的结果。

该方法注意事项如下：

  • 线程任务类只是实现接口，可以继续继承类和实现接口，扩展性强，但是编码麻烦。
  • 可以在线程执行完毕后去获取线程执行的结果。
  • 通过 get 方法去获取任务执行的结果，虽然该语句所在线程与子线程是并行的，但是会等到子线程执行完毕有返回值后才执行。

4. 线程同步

线程同步是为了解决线程安全问题，通过 synchronized 把共享资源进行上锁，每次只能一个线程进入访问完毕以后解锁，然后其他线程才能进来。JDK5 以后提供了一个新的锁对象 Lock，更加灵活、方便。

4.1 同步代码块

同步代码块的概述

同步代码块即使用 synchronized 修饰代码块，包括实例方法中的代码块和静态方法中的代码块。

格式为：synchronized(同步锁对象) { 操作共享资源的代码(核心代码) }。

对于当前同时执行的线程来说，当锁对象是同一个对象，则只允许一个线程进入，若为不同对象，则不设限。

同步代码块的使用

规范上建议使用共享资源作为锁对象：

  • 实例方法中的代码块：建议使用 this 作为锁对象。
  • 静态方法中的代码块：建议使用类名.class 对象作为锁对象。

4.2 同步方法

同步方法的概述

同步方法即使用 synchronized 修饰方法，包括实例方法和静态方法。

格式为：修饰符 synchronized 返回值类型 方法名称(形参列表) { 操作共享资源的代码(核心代码) }。

对于当前同时执行的线程来说，当锁对象是同一个对象，则只允许一个线程进入，若为不同对象，则不设限。锁是可以锁多个方法的，只要这些方法中锁对象一样。

同步方法的使用

同步方法其实底层也是有隐式锁对象的，只是锁的范围是整个方法代码：

  • 实例方法：同步方法默认使用 this 作为锁对象。
  • 静态方法：同步方法默认使用类名.class 对象作为锁对象。

6. 线程池

线程池就是一个可以复用线程的技术。如果用户每发起一个请求，后台就创建一个新线程来处理，下次新任务来了又要创建新线程，而创建新线程的开销是很大的，这样会严重影响系统的性能。

6.1 创建线程池

线程池的创建

JDK5 起提供了代表线程池的接口 ExecutorService。

得到线程池有以下两种方式：

  • 方式一：使用 ExecutorService 的实现类 ThreadPoolExecutor 自创建一个线程池对象。

  • 方式二：使用 Executors（线程池的工具类）调用方法返回不同特点的线程池对象。

线程池的特点

线程池中的两个关键的时间点如下：

  • 创建临时线程：新任务提交时发现核心线程都在忙，任务队列也满了，并且还可以创建临时线程，此时才会创建临时线程。
  • 开始拒绝任务：核心线程和临时线程都在忙，任务队列也满了，新的任务过来的时候才会开始任务拒绝。

类的方法

Executors 得到线程池对象常用方法如下：

方法名称 方法说明

public static ExecutorService newFixedThreadPool(int nThreads) 创建固定线程数量的线程池

Executors 的底层其实也是基于线程池的实现类 ThreadPoolExecutor 创建线程池对象的。

类的案例

使用风险

大型并发系统环境中使用 Executors 如果不注意可能会出现系统风险：

  • newSingleThreadExecutor() 和 newFixedThreadPool(int nThreads) ：

    允许请求的任务队列长度是 Integer.MAX_VALUE，可能出现 OOM 错误，即队列中任务太多，内存溢出（ java.lang.OutOfMemoryError）。

  • newCachedThreadPool() 和 newScheduledThreadPool(int corePoolSize)：

    创建的线程数量最大上限是 Integer.MAX_VALUE，线程数可能会随着任务1:1增长，也可能出现OOM错误，内存溢出（ java.lang.OutOfMemoryError ）

8. 并发与并行

正在运行的程序就是一个独立的进程， 线程是属于进程的，多个线程其实是并发与并行同时进行的。

并发

CPU 同时处理线程的数量有限，CPU 会轮询为系统的每个线程服务，每个时刻都只执行着一个线程，但由于 CPU 切换的速度很快，给我们的感觉这些线程在同时执行，这就是并发。类似于三体里锁死人类基础科学的智子。

并行

在同一个时刻上，同时有多个线程在被 CPU 处理并执行。

9. 线程生命周期

Java 总共定义了六种线程状态，六种状态都定义在 Thread 类的内部枚举类中，如下所示：

线程状态 状态说明
New(新建) 线程刚被创建，但是并未启动
Runnable(可运行) 线程已经调用了 start() 等待 CPU 调度
Blocked(锁阻塞) 线程在执行的时候未竞争到锁对象，则该线程进入 Blocked 状态
Waiting(无限等待) 一个线程进入 Waiting 状态，
另一个线程调用 notify 或者 notifyAll 方法才能够唤醒
Timed Waiting(计时等待) 同 waiting 状态，有几个方法有超时参数，调用他们将进入 Timed Waiting 状态。
带有超时参数的常用方法有 Thread.sleep, Object.wait
Teminated(被终止) 因为 run 方法正常退出而终止，或者因为没有捕获的异常终止了 run 方法而终止
