基于 Fluid 主题基础功能修改

1. 前言

从准备搭建个人博客到今天，陆陆续续也快过去接近一个月了。刚好现在正是国庆假期，于是给自己放个小假，做点自己想做的事情。这两天又断断续续弄了一些功能，打算还是简单整理一下给博客做的美化，也当是个备份。

因为之前完全没有接触过前端的东西，所以除了按照 Fluid 主题官方的使用文档和官网提供的功能进行基本配置以外，想要自己添加一些其他的内容对我来说就显得特别困难了。基本上就是到处去看别人的实现代码，看看页面的源代码，大量阅读然后再自己复制一些看看效果，查一些标签的用法，等大致上能弄懂了后就根据自己的需求修改。整个过程确实艰难，如果不是我对于美观有着惊人的执著，大概我就放弃了，等相关的知识都学了再来。现在这样跟盲人过河似的倒也确实学了一些写了一些，就当是提前预习好了。

博客框架使用的是 Hexo，主题是 Fluid。代码大概还有很多不规范的地方，有些思路也比较诡异，等以后正式学习后应该会再整理一遍。

css 文件都一律放在了 Blog\source\css\fluid 路径下，然后按照 Fluid 主题官方文档中的方式进行引入，不会修改源码。js 文件一部分是新建文件放在主题的文件夹下，路径都会标明，除此之外还有一小部分就是直接修改主题的源码了。

4.3 设置表格样式

设置表格居中

主题本身好像就不支持表格居中，所以一直没弄成。后面换了一种思路，就是把表格直接全部拉长到和页面齐宽。

为了实现这一点，在笔记中写表格的时候，就先设置好了每一列的宽度，加起来等于100%。但这样设置好之后，网页上的表格仍然无法撑满整个页面宽度。最后试了各种各样的方式，还是找到了一个可行的办法，只是还没有弄懂是什么原理。

需要使用 html 创建表格，并多创建一列，将多的一列的 margin 和 padding 全部设置成0，相当于是完全隐身的。在文章中用这种形式写表格，也不会影响到我在本地 Typora 上的阅读。同时加上 class="myform" 是为了防止后续修改样式时修改到代码块的样式，主题里貌似不少元素是两边通用的。

设置表格样式

我比较喜欢 Typora 主题中表格样式，所以又自己设定了一下样式，相关的伪类选择器也是在这一部分稍微了解了以下。代码中注释掉的部分是之前写的无内框的，不过现在还是用有边框的样式了。

除此之外还设置表格在一行显示，显示外的部分可以拖动下方进度条看到，感觉这样比较合我心意。

5. 归档页样式修改

Fluid 主题原本的归档页日期和标题都是靠左显示的，导致右边看起来很空，所以我就想着把日期放到右边去。

这里直接修改了主题的源码，在 Blog\themes\fluid\layout\_partials 路径下的 archive-list.ejs 文件，修改的内容已经标注出来了，就是对换了div 和 time 标签的顺序，让先显示标题再显示时间。

然后要使时间靠右边显示，要设置一下 css 样式 justify-content: space-between，它会把子元素靠边对齐平均分剩余的空间，这样就达到想要的效果了。

6. 分类页样式修改

在文章中我添加了一个分类用来保存文献阅读的笔记，但是并不希望这个分类在分类页中显示出来，所以设置了不显示。

因为我分类在命名时都会加上序号，所以文献阅读笔记这个分类我没加序号，按照顺序是排在最后一个的，所以用 css 选择器找最后一元素即可。

7. 自定义页面样式

创建自定义的页面直接查看 Fluid 官方文档就好，如果希望自定义页面采用某些已有的样式，可以参考墨水记忆的文章。如果还想进行更细致的修改，可以复制或自己创建一个 layout 文件，并在里面添加想要的格式。

由于我需要代码注入器能单独为这一页添加标签，所以我将我自定义页的 layout 设置成了我指定的名字。

大部分元素的样式设置是跟着归档页一起设置了，详见给归档页添加动画。除此之外，我还将在手机上不需要显示的元素设置了 display: none，主要是一行确实放不下，分行又破坏美观了。
