1. 链表概述

链表是有序的列表，链表的特点是：

  • 链表是以节点的方式来存储,是链式存储
  • 每个节点包含 data 域、next 域（指向下一个节点）。
  • 链表的各个节点不一定是连续存储。
  • 链表分带头节点的链表和没有头节点的链表，根据实际的需求来确定。

单向链表和双向链表的对比：

  • 单向链表查找的方向只能是一个方向。而双向链表可以向前或者向后查找。
  • 单向链表不能自我删除，需要靠辅助节点（即必须查询到前驱位置）。而双向链表则可以自我删除。

2. 单向链表实现

链表中的节点通过自定义一个节点类来实现，类中属性至少包括 data 和 next 两部分。data 用于存储元素数据，next 用于存储下一个节点的地址。

然后自定义一个管理节点的单向链表类，若为带头指针的单向链表，则链表类的属性应包含一个节点类型的头指针。头指针不用于存储 data，仅用于指向首节点。

2.3 链表方法

2.3.1 增加节点

分为头插法、尾插法、任意位置插入三种方法：

  • 头插法：使新节点指向原头节点，然后 head 指向新节点。

  • 尾插法：使用临时节点 temp 从首节点开始遍历找到尾节点，使尾节点指向新节点。

  • 根据索引插入，需要先判断索引是否为0：

    • 若索引为 0，则直接使用头插法。若索引为 size，则直接使用尾插法。
    • 若索引不为 0 或 size，则使用临时节点 temp 找到索引的前驱位置，使新节点指向索引节点，再使前驱节点指向新节点。（顺序：前驱节点 → 新节点 → 原索引节点）

2.3.2 删除节点

删除节点需要先判断索引：

  • 若索引为 0，则直接使 head 指向头节点的下一个节点。
  • 若索引不为 0，则使用临时节点 temp 找到索引的前驱位置，使前驱节点指向索引节点的后一个节点。

2.3.3 修改节点

修改节点需要先判断索引：

  • 若索引为 0，则使新节点指向原头节点的下一个节点 ，再使 head 指向新节点。
  • 若索引不为 0，则使用临时节点 temp 找到索引前驱位置，使新节点指向原节点的下一个节点，再使前驱节点指向新节点。

2.3.4 查询节点

查询指定索引的前驱节点：使用 for 循环向后 index - 1次。

查询指定索引节点：

  • 若索引为0，则直接输出头节点。
  • 若索引不为0，则则使用临时节点 temp 找到索引前驱位置，输出前驱节点的下一个节点。

2.3.5 显示链表

使用临时节点 temp 从首节点向后遍历并输出，遇节点为空时终止遍历。

3. 双向链表实现

链表中的节点通过自定义一个节点类来实现，类中属性至少包括 data 、 next 和 pre 三部分。data 用于存储元素数据，next 用于存储后置节点的地址，pre 用于存储前置节点的地址.

然后自定义一个管理节点的双向链表类，若为带头尾指针的双向链表，则链表类的属性应包含一个节点类型的头指针和尾指针。头指针不用于存储 data，仅用于指向首节点。尾指针不用于存储 data，仅用于指向尾节点。

3.3 链表方法

3.3.1 增加节点

分为头插法、尾插法、任意位置插入三种方法：

  • 头插法：新节点在前，原头结点在后，二者相互指。然后使 head 指向新节点。

  • 尾插法：新节点在后，原尾节点在前，二者相互指。然后使 tail 指向新节点。

  • 根据索引插入，需要先判断索引是否为 0 或 size：

    • 若索引为 0，则直接使用头插法。若索引为 size，则直接使用尾插法。
    • 若索引不为 0 或 size，则使用临时节点 temp 找到索引节点，使前驱节点和新节点互指，再使新节点和索引节点互指。（顺序：前驱节点 → 新节点 → 原索引节点）

3.3.2 删除节点

删除节点需要先判断索引：

  • 若索引为 0，则初始化头节点的后一节点的 pre，并让 head 指向头节点的后一节点。
  • 若索引为 size - 1，则初始化尾节点的后一节点的 next，并让 tail 指向尾节点的前一节点。
  • 若索引不为 0 或 size - 1，则使用临时节点 temp 找到索引位置，使索引节点的前一节点和后一节点互指。

3.3.3 修改节点

修改节点需要先判断索引：

  • 若索引为 0，则使新节点和原头节点的后一个节点互指 ，再使 head 指向新节点。
  • 若索引为 size - 1，则使新节点和原尾节点的前一个节点互指，并让 tail 指向尾节点。
  • 若索引不为 0 或 size - 1，则使用临时节点 temp 找到索引位置，使新节点和索引节点的后一节点互指，再使新节点和索引节点的前一节点互指。

3.3.4 查询节点

使用 for 循环向后推进 index 次，即找到查询的节点。

3.3.5 显示链表

使用临时节点 temp 从首节点向后遍历并输出，遇节点为空时终止遍历。
