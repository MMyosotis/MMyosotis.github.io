2. 特殊文字

因为笔记中有一些想要重点强调的部分，代码块也不喜欢用灰扑扑的默认样式，所以自己设定了一下。

在 markdown 文章中用标签包裹住内容：<font class="word-stress">内容</font> ，之后渲染成网页后就能看到效果了。

3. 表格样式

3.1 设置表格居中

主题本身好像就不支持表格居中，所以一直没弄成。后面换了一种思路，就是把表格直接全部拉长到和页面齐宽。

为了实现这一点，需要在笔记中写表格的时候，先设置好了每一列的宽度，加起来等于100%。但这样设置好之后，网页上的表格仍然无法撑满整个页面宽度。最后试了各种各样的方式，找到了一个可行的办法，只是还没有弄懂是什么原理。

需要使用 html 创建表格，并多创建一列，将多的一列的 margin 和 padding 全部设置成0，相当于是完全隐身的。在文章中用这种形式写表格，也不会影响到我在本地 Typora 上的阅读。同时加上 class="myform" 是为了防止后续修改样式时修改到代码块的样式。

3.2 设置表格样式

我比较喜欢 Typora 主题中表格样式，所以照着它设置了表格样式，代码中注释掉的部分是之前写的无内框的留作备份，不过现在还是用有边框的样式了。

除此之外还设置表格在一行显示，显示外的部分可以拖动下方进度条看到，这样就避免了换行显示。

7. 进场动画

7.1 目录进场

文章页的目录设置了一个从左向右的入场动画，设置了 animation-delay: 0.5s ，会比文章页面整体进入要晚一点。

7.2 手机端进场

手机端为了流畅就没有设置页面进入等相关动画，取而代之的是增加了文字进入动画，通过 css 媒体查询 @media 来实现多端设置。

7.3 标题进场

文章页的标题没有使用打字机效果，所以增加了一个从上到下的入场动画，和页面整体入场是同时进行的。

8. Tabs 标签

这里使用了插件 Hexo-tag-common ，插件的使用教程按照官方给的来就行，这里针对该标签进行了样式适配。

9. 页边距

Fluid 主题默认的页边距有点宽了两边空白很多，所以最后修改了页边距，并将整体文章宽度往左侧拉长了，这样能显示更多的内容。这里参考了 Gilgamesh’s Blog 的一篇文章。

左边有很大的空隙，所以把页面整体往左加宽了，后面换了高分屏之后又补充了修改。
