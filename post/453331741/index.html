多表查询 - MySQL

1. 多表关系

一对多

一对多关系详见外键约束。

多对多

实现多对多关系需要建立第三张中间表，中间表至少包含两个外键，分别关联两方主键。

一对一

一对一关系多用于单表拆分，将一张表的基础字段放在一张表中，其他详情字段放在另一张表中，以提升操作效率。

在任意一方加入外键，关联另外一方的主键，并且设置外键为唯一的（unique）。

2. 连接查询

2.1 交叉连接

语法

交叉连接查询数据时，会给出多表的所有数据的组合情况（笛卡尔积）。即表1有n行，表2有m行，则查询结果为n×m行数据：

  • 交叉连接查询：当为表起了别名后，采用表名.字段来选定字段时，表名处只能填入别名

    select 字段列表 from 表1 [表1别名], 表2 [表2别名];

案例

后续其他章节的查询案例，均使用和下列相同的表格，不再单独列出。

2.2 内连接

语法

内连接查询的是两张表交集的部分，如下图所示：

  • 隐式内连接查询

    select 字段列表 from 表1, 表2 where 条件列表;

  • 显式内连接查询

    select 字段列表 from 表1 [inner] join 表2 on 连接条件;

案例

2.3 外连接

语法

外连接查询的是左/右表的所有数据，以及两张表交集部分数据，如下图所示：

  • 左外连接查询：左指的是 left join 关键字的左边的表

    select 字段列表 from 表1 left [outer] join 表2 on 连接条件;

  • 右外连接查询：右指的是 right join 关键字的右边的表

    select 字段列表 from 表1 right [outer] join 表2 on 连接条件;

  • 满外连接查询：MySQL 不支持

案例

2.4 自连接

语法

自连接查询的两张表为同一张表，需要起两个不同的别名，即 表1 别名1 和 表1 别名2。可以是内连接查询，也可以是外连接查询。

3. 联合查询

语法

对于 union 联合查询，就是把多次查询的结果合并起来，形成一个新的查询结果集。

注意

  • union all 会将全部的数据直接合并在一起，union 会对合并之后的数据去重。
  • 对于联合查询的多张表字段的列数必须保持一致，字段类型也需要保持一致。

4. 子查询

4.1 子查询概述

SQL 语句中嵌套 select 语句，称为嵌套查询，又称为子查询。

子查询外部的语句可以是 insert/update/delete/select 的任何一个，子查询的位置可以为 where/from/select 之后。

根据子查询结果的不同，可以分为以下几类：

  • 标量子查询：子查询结果为单个值
  • 列子查询：子查询结果为一列
  • 行子查询：子查询结果为一行
  • 表子查询：子查询结果为多行多列

4.2 标量子查询

子查询返回的结果是单个值，常用操作符为：=、<>、>、>=、<、<=

4.3 列子查询

操作符

子查询返回的结果是一列，常用操作符如下：

操作符 描述
in 在指定的集合范围之内，多选一
not in 不在指定的集合范围之内
any 子查询返回列表中，有任意一个满足即可
some 与 any 等同
all 子查询返回列表的所有值都必须满足

案例

子查询返回的结果是一行，常用的操作符为：=、<>、in、not in
