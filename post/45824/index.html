常用 API - Java SE

1. Object 类

Java 中一个类要么默认继承了 Object 类，要么间接继承了 Object 类。

Object 作为所有类的父类，提供了很多常用的方法给每个子类对象拿来使用，其中常用的两个方法：

构造方法 方法说明
public String toString() 默认是返回当前对象在堆内存中的地址信息:类的全限名@内存地址
public boolean equals(Object o) 默认是比较当前对象与另一个对象的地址是否相同，相同返回 true，不同返回 false

1.1 toString 重写

Object 中 toString() 方法默认返回地址，但通过被子类重写，则可以返回对象的内容信息。

在用 System.out 打印对象时，会默认调用对象的 toString() 方法，所以此时可以省略该方法，只写对象。

1.2 equals 重写

Object 中 equals 方法默认比较地址，比较地址时与 == 效果相同。但通过被子类重写，则可以比较对象的内容信息是否一致。

2. Objects 类

Objects 是一个工具类，继承于 Object 类，提供了一些方法去完成一些功能：

方法名称 方法说明
public static boolean equals(Object a, Object b) 比较两个对象的，底层会先进行非空判断，从而可以避免空指针异常，再进行 equals 比较
public static boolean isNull(Object obj) 判断变量是否为 null，为 null 返回 true，反之返回 false

Objects 中 equals 其内部仍然是调用了类自身的 equals 方法，但在调用前会先进行非空判断，避免了空指针异常。

3. String 类

3.1 创建对象

类的概述

java.lang.String 类代表字符串，String 类定义的变量可以用于指向字符串对象，为引用类型。

String 被称为不可变字符串类型，它的对象在创建后不能被更改，String 变量每次的修改其实都是产生并指向了新的字符串对象。

创建类对象

  • 可以通过直接赋值来创建 String 对象：

    直接定义字符串对象，在字符串常量池中存储，而且相同内容只会在其中存储一份。

  • 通过 String 类中构造方法创建对象：

    构造方法 方法说明
    public String() 创建一个空白字符串对象，不含有任何内容
    public String(String original) 根据传入的字符串内容，来创建字符串对象
    public String(char[] chs) 根据字符数组的内容，来创建字符串对象
    public String(byte[] chs) 根据字节数组的内容，来创建字符串对象

    通过构造方法创建对象，每一次都会产生一个新对象，放在堆内存中。

字符集解码

字符串的构造方法同时可以用于字符集的解码。

3.2 比较对象

引用类型用 == 比较是比较的地址值，若是比较内容，需要调用 String 类中的方法：

方法名称 方法说明
public boolean equals (Object anObject) 将此字符串与指定对象进行比较。只关心字符内容是否一致
public boolean equalsIgnoreCase (String anotherString) 将此字符串与指定对象进行比较，忽略大小写比较字符串。只关心字符内容是否一致

3.3 常用方法

字符串拼接

String 和 char 字符拼接：可以直接采用 + 号拼接，参见算数运算符。

查询方法

方法名称 方法说明

public boolean startWiths(String prefix) 判断字符串中是否以指定字符串开始

截取方法

方法名称 方法说明

public String substring(int beginIndex) 从传入的索引处截取，截取到末尾，得到新的字符串

替换和分割方法

方法名称 方法说明

字符串指定规则切割可以参照正则表达式。

4. StringBuilder 类

类的概述

StringBuilder 是一个可变的字符串的操作类，操作字符串的性能比 String 要更高（如拼接、修改等）。

StringBuilder 的构造方法如下：

方法名称 方法说明
public StringBuilder() 创建一个空白的可变的字符串对象，不包含任何内容
public StringBuilder(String str) 创建一个指定字符串内容的可变字符串对象
**类的方法**
方法名称 方法说明
public StringBuilder append(E e) 添加数据并返回 StringBuilder 对象本身
public StringBuilder reverse() 将对象的内容反转并返回 StringBuilder 对象本身
public int length() 返回对象内容长度
public String toString() 把 StringBuilder 转换为 String

对于 append 方法和 reverse 方法支持链式编程。

类的拼接性能

StringBuilder 拼接字符串时性能要优于 String，可以通过内存来具体解释：

  • StringBuilder 中使用 append 方法拼接，只创建了一个对象。

5. Math 类

Math 类包含执行基本数字运算的方法，Math 类没有提供公开的构造器，不能被实例化。

Math 类中的常用方法如下：

方法名称 方法说明
public static int abs(int a) 获取参数绝对值
public static double ceil(double a) 向上取整
public static double floor(double a) 向下取整
public static int round(float a) 四舍五入
public static int max(int a, int b) 获取两个 int 值中的较大值
public static double pow(double a, double b) 返回 a 的 b 次幂
public static double random() 返回 [0.0, 1.0) 之间的 double 类型随机值

关于 random() 方法有如下注意事项：

  • 使用 random() 方法还可以生成指定范围 [a, b) 的随机数，语法格式如下：Math.random() * (b - a) + a 。
  • 也可以生成指定范围 [a, b) 的 int 类型随机数，语法格式如下：(int)(Math.random() * (b + 1 - a) + a) 。

6. Random 类

类的概述

Random 类用于生成伪随机数流，并提供了生成不同类型随机数的方法。

Random 的构造方法如下：

方法名称 方法说明
public Random() 以系统自身的时间为种子来构造 Random 对象
public Random(long seed) 以自己来选定具体的种子来构造 Random 对象

种子设定的注意事项如下：

  • 如果不给定种子，就使用系统当前时间戳作为种子，因此每次运行时种子不同，得到的随机数序列就不同。
  • 但若自己给定相同的种子（seed），则得到的随机数序列是完全确定的。

类的方法

方法名称 方法说明
public int nextInt() 返回值为整数，范围是 int 类型范围
public long nextLong() 返回值为长整型，范围是 long 类型的范围
public float nextFloat() 返回值为 float 小数，范围是 [0.0, 1.0)
public double nextDouble() 返回值为 double 小数，范围是 [0.0, 1.0)
public double nextGaussian() 返回值为呈高斯（正态）分布的 double 值，其平均值是 0.0，标准差是 1.0
public boolean nextBoolean() 返回值为 boolean 值，true 和 false 概率相同

7. System 类

System 类包含了系统级的很多属性和控制方法，System 类没有提供公开的构造器，不能被实例化。

public static long currentTimeMillis() 返回当前系统的时间毫秒值形式
public static void arraycopy(Object src, int srcPos,
Object dest, int destPos, int length)		 数组拷贝，里面参数依次是数据源数组，起始索引，目的地数组，起始索引，拷贝个数

  • 使用 System.exit(0) 可以直接正常终止 JVM，在其之后的代码均不会运行。

  • 时间毫秒值：指的是从1970年1月1日 00:00:00（C语言诞生时间）走到此刻的总的毫秒数。1s = 1000ms。

8. BigDecimal 类

类的概述

BigDecimal 类提供了用于解决浮点型运算精度失真的问题，浮点型四则运算均可存在精度失真。

BigDecimal 虽然存在构造方法 BigDecimal(double)，但是存在精度损失风险，不推荐使用。

创建 BigDecimal 对象推荐使用 valueOf 方法：

方法名称 方法说明
public static BigDecimal valueOf(double val) 可以将浮点数包装成 BigDecimal 对象
**类的方法**
方法名称 方法说明
public BigDecimal add(BigDecimal b) 加法
public BigDecimal subtract(BigDecimal b) 减法
public BigDecimal multiply(BigDecimal b) 乘法
public BigDecimal divide(BigDecimal b) 除法
public BigDecimal divide(BigDecimal b) 除法
public BigDecimal divide(BigDecimal divisor, int scale, int roundingMode) 除法，后两个参数为精确位数、舍入模式

类的注意事项

特别注意 BigDecimal 类型只能进行精度运算，对于无法取到精确值的运算会直接抛出异常 ArithmeticException。

9.3 Calendar 类

类的概念

Calendar 是 java.util.Calendar 包下的代表了系统此刻日期对应的日历的类。Calendar 是一个抽象类，不能直接创建对象。calendar是可变日期对象，一旦修改后其对象本身表示的时间将产生变化。

类的方法

Calendar 创建日历对象的方法：

方法名称 方法说明
public static Calendar getInstance() 获取当前日历对象

Calendar 常用方法如下：

方法名称 方法说明
public int get(int field) 取日历中的某个字段信息
public void set(int field,int value) 修改日历的某个字段信息
public void add(int field,int amount) 为某个字段增加/减少指定的值
public final Date getTime() 拿到此刻日期对象
public long getTimeInMillis() 拿到此刻时间毫秒值

参数 int field 不同数字对应了日历中不同字段，也可以直接通过 Calendar.字段名 来获取字段对应的数字。

9.4 JDK8 新增类

从 JDK8 开始，java.time 包提供了新的日期和时间 API，新增的类有如下特点：

  • 严格区分了时刻、本地日期、本地时间，并且，对日期和时间进行运算更加方便。
  • 类型几乎全部是不变类型（和String的使用类似），可以放心使用不必担心被修改。

9.4.1 日期时间类

类的概述

新增了三个表示日期和时间的类：他们三者构造方法和 API 有共通之处。

  • LocalDate：不包含具体时间的日期。
  • LocalTime：不含日期的时间。
  • LocalDateTime：包含了日期及时间。

其构造方法如下：其中关于时间的参数可以是三项也可以是四项（多一项为纳秒）。

方法名称 方法说明 方法调用
public static 类名 now() 根据当前时间创建对象 LocaDate ld = LocalDate.now();
LocalTime lt = LocalTime.now();
LocalDateTime ldt = LocalDateTime.now();
public static 类名 of(…) 指定日期/时间创建对象 LocalDate ld = LocalDate.of(2099, 11, 11);
LocalTime lt = LocalTime.of(11, 11, 11);
LocalDateTime ldt = LocalDateTime.of(2020, 10, 6, 13, 23, 43);

  • 获取关于日期的信息：只能在 LocalDate 和 LocalDateTime 中调用。

    public Month getMonth() 获取月份（月份的英文全称）

    Public int getDayOfMonth() 获取当月中第几天
    Public int getDayOfYear() 获取当年中第几天

  • 获取关于时间的信息：只能在 LocalTime 和 LocalDateTime 中调用。

    方法名称 方法说明
    public int getHour() 获取时
    public int getMinute() 获取分
    public int getSecond() 获取秒

9.4.2 Instant 时间戳

Instant 类由一个静态的工厂方法 now() 可以返回当前时间戳，包含了日期和时间。注意事项如下：

  • 在不指定时区时，返回的是世界标准时间。若要指定时区，需要调用类中的 atZone() 方法。
  • Instant 和 Date 这两个类可以进行转换。

9.4.3 DateTimeFormatter 类

在 JDK8 中，引入了一个全新的日期与时间格式器 DateTimeFormatter 。该类与 SimpleDateFormat 类的最大区别在于，其正反均可调用 format 方法，但无法格式化 Date 类的日期时间，只能格式化实现了 TemporalAccessor 接口的对象。

DateTimeFormatter 无法使用构造方法创建对象，应当使用 ofPattern 方法：

方法名称 方法说明
public static DateTimeFormatter ofPattern(String pattern) 创建日期格式化对象，并封装格式化的形式信息

9.4.4 日期时间差

日期差和时间差分别用两个类来计算：

  • 计算日期差：在 JDK8 中，我们可以使用 Period 类来计算日期间隔差异，用于 LocalDate 之间的比较。

    通过Period 类方法 getYears()、getMonths()、getDays() 来计算，只能精确到年月日。

  • 计算时间差：在 JDK8 中，我们可以使用 Duration 类来计算时间间隔差异，用于 LocalDateTime、LocalTime、或 Instant 之间的比较。

    通过Duration 类方法 toDays()、toHours()、toMinutes()、toMillis()、toNanos() 来计算。
