JDBC - MySQL

1. 基本概述

JDBC 就是使用 Java 语言操作关系型数据库的一套 API。

  • 本质是是官方定义的一套操作所有关系型数据库的规则，即接口。

  • 各个数据厂商去实现这套接口，提供数据驱动 jar 包；我们可以使用 jar 包中的实现类。

现在提供一个简单的连接使用案例。

2. DriverManager

DriverManager 是驱动管理类，用于注册驱动、获取数据库连接。

注册驱动

  • 注册驱动：MySQL 5之后的驱动包可以省略注册驱动的步骤

    Class.forName("com.mysql.jdbc.Driver");

方法

DriverManager 中获取连接的方法如下：

方法返回值 方法名称 方法说明
Static Connection getConnection(String url, String user, String password) 获取连接

其中参数含义如下：

  • url：连接路径，格式为 jdbc:mysql://主机名:端口名/数据库名?参数1=值1&参数2=值2...
    • 如果连接的是本机 MySQL 服务器，并且 MySQL 服务器默认端口是3306，则可以简写为 jdbc:mysql:///数据库名?参数1=值1&参数2=值2...
  • user：用户名
  • password：用户密码

3. Connection

Connection 是数据库连接类，用于获取执行 SQL 的对象、管理事务。可以通过 DriverManager 的方法获取。

方法

Connection 中获取执行 SQL 的对象的方法如下：

方法返回值 方法名称 方法说明
Statement createStatement() 普通执行 SQL 对象
PreparedStatement prepareStatement(String sql) 预编译 SQL 的执行 SQL 对象，并防止 SQL 注入
CallableStatement prepareCall(String sql) 执行存储过程的对象

Connection 中事务管理的方法如下：此处参考 MySQL 的事务管理基本操作。

方法返回值 方法名称 方法说明
void setAutoCommit(boolean autoCommit); true 为自动提交事务；false 为手动提交事务，即开启事务；默认为true
void commit() 提交事务
void rollback() 回滚事务

4. Statement

Statement 是执行 SQL 语句的类，用于执行 SQL 语句。可以通过 Connection 的方法获取。

方法

Statement 中执行 SQL 语句的方法如下：

方法返回值 方法名称 方法说明
int executeUpdate(String sql) 执行 DML（返回值为受影响的行数）、DDL（返回值均有可能）
ResultSet executeQuery(String sql) 执行 DQL 语句，返回结果集 ResultSet 对象

案例

5. ResultSet

ResultSet 是结果集，用于封装 DQL 语句的查询结果。可以通过 Statement 的方法获取。

方法

ResultSet 中获取查询结果的方法如下：

方法返回值 方法名称 方法说明
boolean next() 移到下一行，返回值为 true 则表示当前行有数据，false 则表示没有数据
xxx xxx getXxx(String columnLabel) 获取名称为 str 的列的数据
xxx xxx getXxx(int columnIndex) 获取第 num 列的数据（从1开始）

6. PreparedStatement

PreparedStatement 是执行 SQL 语句的类，用于执行 SQL 语句（可以预编译和防止 SQL 注入）。可以通过 Connection 的方法获取。

6.1 登录案例

以下案例提供了一种登录逻辑：根据用户输入的用户名和密码在数据库进行查找，若查找到则视为登录成功。

6.2 SQL 注入

SQL 注入是通过操作输入来修改事先定义好的 SQL 语句，用以达到执行代码对服务器进行攻击的方法。

6.3 PreparedStatement

概述

PreparedStatement 继承了 Statement，是执行 SQL 语句的类。

  • 可以预编译 SQL：预编译功能开启需要在 url 里设置 useServerPrepStmts=true
  • 可以防止 SQL 注入：将敏感字符进行转义，字符串 "' or '1' = '1" 会被转义成 "\' or \'1\' = \'1"，称为纯字符串

方法

在 Connection 中提供了获取 PreparedStatement 对象的方法， 其中传入的 SQL 语句中的参数值使用占位符 ? 替代。

PreparedStatement 对象设置参数值的方法如下：

方法返回值 方法名称 方法说明
void setXxx(int parameterIndex, xxx x) parameterIndex 为 ? 的位置编号（从1开始），x 为替换 ? 的值

PreparedStatement 中执行 SQL 语句的方法如下：

方法返回值 方法名称 方法说明
int executeUpdate() 执行 DML（返回值为受影响的行数）、DDL（返回值均有可能）
ResultSet executeQuery() 执行 DQL 语句，返回结果集 ResultSet 对象

7. 数据库连接池

数据库连接池是个容器，负责分配、管理数据库连接（Connection）：

  • 它允许应用程序重复使用一个现有的数据库连接，而不是再重新建立一个
  • 释放空闲时间超过最大空闲时间的数据库连接来避免因为没有释放数据库连接而引起的数据库连接遗漏

官方提供了数据库连接池标准接口 DataSource，由第三方组织实现此接口，并重写对应的 getConnection() 方法，用于获取连接 Connection。常见的数据库连接池：DBCP、C3P0、Druid（相关介绍以及参数配置详见其 Github 地址：https://github.com/alibaba/druid）。

步骤

普通配置并连接 Druid 的步骤如下：

  • 导入对应的 jar 包

  • 在文件中进行属性配置并导入配置文件，具体属性详见 Druid 配置属性列表

  • 获取数据库连接池对象 DataSource

  • 获取连接 Connection
