面向对象 (一) - Java SE

1. 三大特性

  • 封装：指的是类的内部信息进行隐藏，一般是指对类内部的属性进行私有化，外部无法直接访问或影响内部的属性，只能通过特定的方法对封装的内容进行访问，提高了代码的安全性。
  • 继承：指子类继承父类，子类可以获得父类的全部属性和方法，且可以拥有一些父类所没有的方法和属性，提高了代码的复用性和逻辑性。
  • 多态：多态性是指同一个操作用于某一对象，可以有不同的解释，产生不同的执行结果。

2. 类和对象

2.1 类的定义

定义类

一个 Java 文件中可以定义多个 class 类，但只能一个类是 public 修饰，而且 public 修饰的类名必须成为代码文件名，实际建议一个 java 文件定义一个 class 类。类名通常采用大驼峰命名法。

类的分类

  • Javabean 类：用来描述一类事物的类。
  • 测试类：用来检查其他类是否书写正确，带有 main 方法的类，是程序的入口。
  • 工具类：不用于描述一类事物，而是用于实现一些方法的类。

2.2 访问对象

访问对象

创建的 Animals 对象存放于堆中，同时堆中还存放一个指向方法区中成员方法的引用地址。

对象变量 animal 存放于栈中为引用数据类型，存放的是指向堆中对象的地址：

访问成员变量

  • 静态变量访问：可以采用类名和对象名去访问静态成员变量。
  • 实例变量访问：只能创建对象并用对象名去访问。

访问成员方法

  • 静态方法访问：可以采用类名和对象名去访问静态成员变量。
  • 实例方法访问：只能创建对象并用对象名去访问。

2.3 this 关键字

this 关键字用于方法和构造器中，代表当前对象的地址值，可以用于指定访问当前对象的成员和构造方法。

当用 this() 访问构造方法时，必须要放在方法体内的第一行。

3. 构造方法

定义构造方法

构造方法又名构造器，定义在类中，可以用于初始化一个类的对象，并返回对象的地址。

构造方法的参数

构造方法分为有参数构造方法和无参数构造方法：

  • 无参数构造方法：初始化对象时，成员变量均采用数据类型的默认值。若未定义任何构造方法，则存在一个默认的无参数构造方法。
  • 有参数构造方法：初始化对象时，同时可以接受参数为对象赋值。

4. 成员方法

4.1 方法定义

方法的格式

方法的完整格式包括修饰符、返回值类型、方法名称、形参列表、执行代码几个部分。方法名通常采用大驼峰命名法。

对于不需要返回值的方法，返回值类型必须声明为 void，此时方法内部不可以用 return 返回数据。

方法的内存图

方法在内存中存放于方法区，调用方法时，方法运行区域为栈内存。

4.2 参数传递

4.2.1 值传递

在传输实参给方法的形参的时候，并不是传输实参变量本身， 而是传输实参变量中存储的值，这就是值传递：

  • 基本类型的参数传输存储的数据值。
  • 引用类型的参数传输存储的地址值。

4.2.2 可变参数

可变参数在形参中非常灵活，方便。可以不接收参数，可以接收1个或者多个参数，也可以接收一个数组。

可变参数的使用注意事项是：

  • 一个形参列表中可变参数只能有一个。
  • 可变参数必须放在形参列表的最后面。
  • 可变参数传入方法后，在方法内部即是一个数组，可调用数组的方法。

4.3 方法重载

同一个类中出现多个方法名称相同，但是形参列表（类型、顺序、数量）是不同的，不论返回类型和修饰符，这些方法就是重载方法。

调用重载方法

调用方法的时候，会通过参数的不同来区分调用的是哪个方法。

5.3 包的使用

创建包

包是用来分门别类的管理各种不同类的，类似于文件夹。

建包语句必须放在第一行，包名格式建议：公司域名倒写.技术名称，建议全部小写。

导入包

相同包下的类可以直接访问，不同包下的类必须导包才可以使用。但使用 java.lang 包中的类时不需要导包。

注意

不同包下的同名类：假如一个类中需要用到不同类，而这个两个类名称是一样的，则默认只能导入一个类，另一个类要带包名访问。
