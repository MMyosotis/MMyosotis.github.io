1. 基本概述

概述

Spring Framework 的系统架构如下图所示：

IoC (Inversion of Control) 控制反转：

  • 使用对象时，由主动 new 产生对象转换为由外部提供对象，此过程中对象创建控制权转移到外部，即控制反转。
  • Spring 提供了一个 IoC 容器，用来充当 IoC 思想中的”外部“。
  • IoC 容器负责对象的创建、初始化等一系列工作，被创建或背管理的对象在 IoC 容器中统称为 Bean。

DI (Dependency Injection) 依赖注入：

  • 在容器中建立 bean 与 bean 之间的依赖关系的整个过程，称为依赖注入

5. bean 生命周期

bean 可以通过配置来指定生命周期的控制方法。

5.1 生命周期

概述

  • 初始化容器：创建对象（内存分配） -> 执行构造方法 -> 执行属性注入 -> 执行 bean 初始化方法
  • 使用 bean：执行业务操作
  • 关闭容器：执行 bean 销毁前的方法

注意

若不关闭容器而是直接结束程序，则程序结束后会直接关闭虚拟器，不再执行关闭容器的操作，也就无法执行 bean 销毁前的方法。

5.2 自定控制方法

语法

6. 依赖注入

bean 可以通过配置来指定不同的方式进行依赖注入。

依赖注入方式的选择：

  • 强制依赖建议使用构造器进行，使用 setter 注入有概率不进行注入导致null对象出现
  • 可选依赖建议使用 setter 注入
  • 自己开发的模块推荐使用 setter 注入

6.1 Setter 注入

6.1.1 引用类型

语法

    在 bean 中使用 property 标签的 ref 属性注入引用类型对象

案例

6.1.2 基本类型

语法

    在 bean 中使用 property 标签的 value 属性注入简单类型数据

案例

6.1.3 集合注入

语法

    在 bean 中使用 property 标签下的集合标签注入对象：

    标签 说明 注入基本类型 注入引用类型
    <array> 数组 <value>值</value> <refer bean="注入的bean的id"/>

    <properties> properties 集合 <prop key="key值">value值</prop>

6.2.3 参数适配

除了在上述案例中提到的用 constructor-arg 标签的 name 属性去对应构造方法中的形参以外，还有两种适配方法

  • 使用 type 属性按形参类型注入，将根据类型区分是哪个形参

  • 使用 index 属性按形参位置注入，将根据位置区分是哪个形参（从0开始数）

6.3 自动装配

IoC 容器根据 bean 所依赖的资源在容器中自动查找并注入到 bean 中的过程称为自动装配。

  • 自动装配用于引用类型依赖注入，不能对简单类型进行操作
  • 使用按类型装配时（byType）必须保证容器中相同类型的 bean 唯一
  • 使用按名称装配时（byName）必须保证容器中相同名字的 bean 唯一，但该方法会导致变量名和配置耦合度高
  • 自动装配优先级低于 Setter 注入与构造器注入，同时出现时自动装配配置失效

6.3.1 按类型

语法

在类中定义引用类型属性并提供可访问的 set 方法，在 bean 中使用 bean 标签的 autowire 属性设置自动装配。

  • 按类型自动装配：类A.类B = 配置文件.类B，则会容器中给 beanA 注入 beanB

案例

  • App.java
  • BookDao.java
  • BookDaoImp.java
  • BookService.java
  • BookServiceImp.java

6.3.1 按名称

语法

在类中定义引用类型属性并提供可访问的 set 方法，在 bean 中使用 bean 标签的 autowire 属性设置自动装配。

  • 按名称自动装配：类A. set 方法后面的名字 = 配置文件.bean名B，则会容器中给 beanA 注入 beanB

9. Ioc 容器

IoC 容器的使用选择：

  • 适合交给容器管理的 bean：表现层对象、业务层对象、数据层对象
  • 不适合交给容器进行管理的 bean：封装实体的域对象

9.1 容器接口

概述

BeanFactory 是容器的最顶层接口，在 Spring 早期使用其实现类作 IoC 容器，现在已经弃用了。

ApplicationContext 是现在常用的容器核心接口，其接口中常用的实现类为：

  • ClassPathXmlApplicationContext：使用类路径加载配置文件
  • FileSystemXmlApplicationContext：使用文件路径加载配置文件

注意

BeanFactory 与现在常用的 ApplicationContext 的区别在于 bean 的加载时机：

  • BeanFactory：延迟加载 bean，即容器对象造出来时 bean 并未被加载，而是使用容器获取 bean 时才加载

  • ApplicationContext：在容器对象造出来时 bean 就会加载，若想实现延迟加载需要配置 bean 的属性 lazy-init

9.2 获取容器

获取容器需要使用 ApplicationContext 下的子类，有以下多种方式：

  • 类路径加载配置文件：使用 ClassPathXmlApplicationContext，配置文件放在类路径下

  • 加载多个配置文件

  • 文件路径加载配置文件：使用 FileSystemXmlApplicationContext，配置文件放在任意路径下

9.3 关闭容器

关闭容器需要使用 ApplicationContext 中实现类 ClassPathXmlApplicationContext 的方法。

使用时遵循多态调用成员原则，左侧的类也必须使用 ClassPathXmlApplicationContext。

  • 手动关闭容器：使用 close() 方法

  • 注册关闭钩子：使用 registerShutdownHook() 方法，可以在虚拟机退出前先关闭容器再退出虚拟机

9.4 获取对象

获取容器中的 bean 有以下多种方式：

  • 使用 bean 名称获取：获取后需要强转类型

  • 使用 bean 名称获取并指定类型

  • 使用 bean 类型获取：要求该类型的 bean 唯一
