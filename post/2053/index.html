泛型 - Java SE

1. 泛型的使用

概念

泛型是 JDK5 中引入的特性，它限制了能添加进集合的类型，可以在编译阶段约束操作的数据类型，并进行检查。实际在运行时类并未收到泛型的约束。

语法

泛型格式为 <数据类型> ，其数据类型只支持引用数据类型。

在未给集合指定任何类型时，默认所有的数据类型都是 Object 类型。

2. 泛型的定义

2.1 泛型类

定义类时同时定义了泛型的类就是泛型类，其原理是把出现泛型变量的地方全部替换成传输的真实数据类型。

2.2 泛型方法

定义方法时同时定义了泛型的方法就是泛型方法，泛型写在修饰符之后，返回类型之前。

在调用泛型方法时，即可以显示指定类型，也可以根据形参类型隐式推断而不必指定。

2.3 泛型接口

使用了泛型定义的接口就是泛型接口，泛型写在接口名之后。泛型接口有两种使用方式：

  • 实现类给出具体的类型。

  • 实现类延续泛型，创建实现类对象时再确定类型。

3. 泛型的特性

泛型具有如下特性：

  • 指定泛型的具体类型后，传递泛型时不具备多态，不能传入其子类类型；传递数据时，可以传入该类类型或者其子类类型。

  • 泛型不能实例化：不能创建对象、创建对应数组等。

4. 泛型通配符

当泛型类型不确定，但又想要指定为某一个继承体系里的类型，则可以使用泛型通配符：

  • <?> ：无边界通配符，可以接受未知的数据类型
  • <? extends E> ：表示可以传递 E 和 E 的所有子类类型。
  • <? super E> ：表示可以传递 E 和 E 的所有父类类型。

5. 泛型擦除

泛型信息只存在于代码编译阶段，在进入 JVM 之前，与泛型相关的信息会被擦除掉，即泛型擦除。

泛型擦除这一性质可以使得反射能绕过泛型向集合中添加元素。
