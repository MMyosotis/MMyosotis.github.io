Spring (二) - SSM 框架

1. bean 基本配置 (注解)

语法

对比不使用注解直接配置 bean，使用注解可以进一步简化开发。

  • 使用 @Component 定义 bean：若不配置 bean 名，则默认使用首字母小写的类名做 bean 名

    配置文件中通过组件扫描加载 bean：需要开启 context 命名空间，详见加载属性集

除了 @Component 以外，Spring 还提供了三个衍生注解，功能与 @Component 一致只是含义不一样：

  • @ Controller：用于表现层 bean 定义
  • @ Service：用于业务层 bean 定义
  • @ Repository：用于数据层 bean 定义

案例

  • App.java
  • BookDaoImp.java

2. Spring 配置文件 (注解)

步骤

对比不使用注解直接在 Spring 配置文件中进行 bean 的配置，可以通过注解使 Java 类替代配置文件，实现纯注解开发。

  • 使用 @Configuration 注解用于设定当前类为配置类

  • 使用 @ComponentScan("扫描包路径") 注解用于设定扫描bean的路径

    此注解只能添加一次，若想设置多个扫描路径则须使用数组格式 @ComponentScan({“路径1”, “路径2”, …})

  • 在主程序中使用 AnnotationConfigApplicationContext 获取容器，其他步骤不变

案例

  • App.java
  • SpringConfig.java
  • BookDaoImp.java

3. bean 作用范围 (注解)

语法

对比不使用注解直接配置 bean 作用范围，使用注解可以进一步简化开发。

  • 使用注解 @Scope 配置单例 singleton|多例 prototype，不配置该项则默认为单例

案例

  • App.java
  • SpringConfig.java
  • BookDaoImp.java
  • UserDaoImp.java

4. bean 生命周期 (注解)

语法

对比不使用注解直接配置 bean 生命周期方法，使用注解可以进一步简化开发。

  • 使用 @PostConstruct 和 @PreDestroy 用于设定当前类的初始化方法和销毁前方法

注意

若不关闭容器而是直接结束程序，则程序结束后会直接关闭虚拟器，不再执行关闭容器的操作，也就无法执行 bean 销毁前的方法。

使用配置类代替配置文件时，在 AnnotationConfigApplicationContext 中关闭容器的方法与 ClassPathXmlApplicationContext 中一致。

案例

  • App.java
  • SpringConfig.java
  • BookDaoImp.java

5. 依赖注入 (注解)

5.1 引用类型

使用注解进行引用类型的依赖注入只提供了自动装配这一种方式，对比不使用注解直接依赖注入，使用注解可以进一步简化开发。

  • 引用类型的自动装配基于反射（即公有和私有均能获取）获取属性并初始化，因此无需提供 setter 方法
  • 自动装配建议使用无参构造器创建对象（默认）
  • 按类型自动装配，要求该类型只能有这一个 bean
  • 按名称自动装配，要求该名字只能有这一个 bean

5.1.1 按类型

语法

  • 按类型自动装配：使用注解 @Autowired 在类A中注入类B

案例

  • App.java
  • SpringConfig.java
  • BookDaoImp.java
  • BookServiceImp.java

5.1.2 按名称

语法

  • 按名称自动装配：使用注解 @Autowired 和 @Qualifier("自动装配的bean名") 在类A中注入类B

案例

  • App.java
  • SpringConfig.java
  • BookDaoImp.java
  • BookServiceImp.java

5.2 基本类型

基本类型的依赖注入基于反射（即公有和私有均能获取）获取属性并初始化，因此无需提供 setter 方法。

语法

  • 使用注解 @Value("注入的值") 在类中实现基本类型的注入

案例

  • App.java
  • SpringConfig.java
  • BookDaoImp.java

6. 管理第三方 bean (注解)

以配置 Druid 为例，区别于正常管理第三方 bean，使用注解可以一定程度上简化配置。

6.1 bean 管理

语法

使用第三方 jar 包时，由于包中的类都是已经写好的，因而无法将所需的类配置成 bean，则需要自己额外定义一个方法去获取所需类的 bean。

使用注解 @Bean 定义 bean：若不配置 bean 名，则默认使用方法做 bean 名。共有以下两种方式管理第三方 bean：

  • 直接在配置类中定义 bean

  • 在独立的类中定义 bean，并将使用注解 @Import(类.class) 将其导入到核心配置类中

    此注解只能添加一次，若想设置多个文件则须使用数组格式 @Import({类1.class, 类2.class, …})

步骤

在 Spring 中配置 Druid 连接的步骤如下：

  • 在 Maven 配置文件中配置相关依赖：mysql、druid
  • 定义获取 bean 的方法并配置相关属性，具体属性详见 Druid 配置属性列表
  • 获取数据库连接池对象 DataSource
  • 获取连接 Connection

案例

  • pom.xml
  • App.java
  • SpringConfig.java

6.2 bean 注入

使用第三方 jar 包时，由于包中的类都是已经写好的，因而无法在所需类中直接进行依赖配置，而需要自己额外定义一个方法去获取所需类的 bean，并在该 bean 中进行依赖配置。

6.2.1 引用类型

语法

  • 将使用按类型自动装配：bean 定义方法的形参类型 = 需要依赖注入的类型

案例

  • pom.xml
  • App.java
  • SpringConfig.java
  • jdbcConfig.java
  • BookDao.java

6.2.2 基本类型

语法

  • 直接在 bean 中使用注解进行基本类型的依赖注入即可。

案例

  • pom.xml
  • App.java
  • SpringConfig.java
  • jdbcConfig.java

7. 加载属性集 (注解)

对于直接写在属性集 Properties 文件中的属性，需要先加载文件然后再使用里面的值，对比不使用注解直接加载属性集，使用注解可以进一步简化开发。

步骤

  • 使用 @PropertySource(classpath:文件.properties) 注解设定 properties 文件路径。

    此注解只能添加一次，若想设置多个文件则须使用数组格式 @PropertySource({“文件1”, “文件2”, …})

    使用注解开发时，文件路径不允许使用 * 占位符

  • 使用读取加载的属性值：${key值}，则会在此处会直接引入 value 值

案例

  • App.java
  • SpringConfig.java
  • BookDaoImp.java
  • example.properties
