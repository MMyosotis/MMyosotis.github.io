Maven - 项目管理

1. 概述

1.1 基本概述

概述

Apache Maven 是一个项目管理和构建工具，它基于项目对象模型（POM）的概念，通过一小段描述信息来管理项目的构建、报告和文档。其特点为：标准化的项目结构、标准化的构建流程、方便的依赖管理。

Maven 项目模型如下：

仓库

Maven 仓库包括：

  • 本地仓库：自己计算机上的一个目录
  • 中央仓库：由 Maven 团队维护的全球唯一的仓库，地址：https://repo1.maven.org/maven2
  • 远程仓库（私服）：一般由公司团队搭建的私有仓库（用于提升访问速度）

仓库使用顺序如下：

  • 当项目中使用坐标引入对应依赖 jar 包后，首先会查找本地仓库中是否有对应的 jar 包，有则直接引用

  • 如果本地仓库没有 jar 包：

    • 没有远程仓库：从中央仓库去寻找对应 jar 包并下载到本地仓库

    • 有远程仓库：从远程仓库中查找对应 jar 包并下载到本地仓库

      ​ 如果远程仓库没有，则会从中央仓库去寻找对应 jar 包下载到远程仓库，再从远程仓库下载到本地仓库

坐标

Maven 中的坐标用于描述仓库中资源的位置，主要组成如下：

  • groupID：定义当前 Maven 项目所属组织名称（通常是域名反写，例如：org.mybatis）
  • artifactID：定义当前 Maven 项目名称（通常是模块名称）
  • version：定义当前项目版本号

1.2 依赖配置

依赖管理

依赖指当前项目运行所需要的 jar 包，一个项目可以在 pom.xml 文件中通过坐标配置多个依赖。

依赖范围

可以通过设置坐标的依赖范围（scope）来设置对应 jar 包的作用范围，默认取 compile。

作用范围分为三类：

  • 编译环境：主程序范围有效，即 main 文件夹范围内
  • 测试环境：测试程序范围有效，即 test 文件夹范围内
  • 运行环境：指运行时，也即参与了打包，即 package 指定范围内
依赖范围 编译环境 测试环境 运行环境 例子
compile √ √ √ logback
test - √ - Junit
provided √ √ - servlet-api
runtime - √ √ JDBC 驱动
system √ √ - 存储在本地的 jar 包
import 引入 DependencyManagement

可以在 pom.xml 文件中配置依赖时指定依赖范围。

1.3 生命周期

Maven 构建项目生命周期描述的是一次构建过程经历了多少个时间，Maven 对项目构建的生命周期划分为三类，各自为一个生命周期：

  • clean：清理工作
    • pre-clean：执行一些需要在 clean 之前完成的工作
    • clean：移除所有上一次构建生成的文件
    • post-clean：执行一些需要在 clean 之后立刻完成的工作
  • default：核心工作，例如编译、测试、打包、安装等，其中常见的命令如下
    • compile：编译项目的源代码
    • test：使用合适的单元测试框架运行测试
    • package：将编译后的代码打包成可分发格式的文件，比如jar、war、ear文件
    • install：安装项目包到本地仓库，这样项目包可以用作其他本地项目的依赖
  • site：产生报告，发布站点等
    • pre-site：执行一些需要在生成站点文件之前完成的工作
    • site：生成项目的站点文档
    • post-site：执行一些需要在生成站点文档之后完成的工作，并且为部署做准备
    • site-deploy：将生成的站点文档部署到特性的服务器上

在同一条生命周期里，执行后面的命令时，该命令前全部命令会先自动执行。

1.4 插件

  • 插件与生命周期内的阶段绑定，在执行到对应生命周期时执行对应的插件功能
  • 默认 Maven 在各个生命周期上绑定有预设的功能
  • 通过插件可以自定义其他功能

官方给出的插件列表：https://maven.apache.org/plugins/index.html

官方给出的插件配置指南：https://maven.apache.org/guides/mini/guide-configuring-plugins.html
