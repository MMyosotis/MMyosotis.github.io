1.4 流程函数

函数

流程函数可以在 SQL 语句中实现条件筛选，从而提高语句的效率

  • 条件函数 if：如果 value 为 true，则返回 t，否则返回 f

    if(value, t, f)

  • 判断为空函数 ifnull：如果 value1 不为 null，返回 value1，否则返回 value2

    ifnull(value1, value2)

  • 分支选择函数 case when

注意

分支选择函数 case when 当一个条件满足，就返回对应的结果，不会继续下一条件判断。

2. 正则表达式

语法

  • 指定正则表达式的字符匹配模式

    字段 regexp '正则表达式'

格式

MySQL 中的正则表达式与 Java 中的正则表达式基本一致，常用匹配方式如下：

模式 描述
^ 匹配输入字符串的开始位置
$ 匹配输入字符串的结束位置
. 匹配除"\n"之外的任何单个字符
[] 字符集合，匹配所包含的任意一个字符
[^] 负值字符集合，匹配未包含的任意字符
| 连接多个匹配条件表示逻辑或
* 其前导字符必须在目标对象中出现零次或连续多次
+ 其前导字符必须在目标对象中连续出现一次或多次
{n} 其前导字符正好出现 n 次
{n,m} 其前导字符至少出现 n 次，但不超过 m 次
**注意**
  • regexp 和 like 的区别为：regexp 只需要匹配上列值中部分字符串，但 like 需要匹配上整个字符串。
  • Java 和 MySQL 的正则表达式的区别：Java 中 matches() 需要匹配上整个字符串，regexp 只需要匹配上列中部分字符串。
  • MySQL 中 \ 为转义符，正则表达式中 \ 也为转义符。
  • 字符串中要输出特殊符号本身必须用 \符号，正则表达式中要输出特殊符号本身必须用 \\符号。
  • 字符串中输出 \ 符号本身必须使用 \\，正则表示中输出 \ 符号本身必须使用 \\\\

3. 约束

约束是作用于表中字段上的规则，用于限制存储在表中的数据，可以在创建表或修改表的时候添加约束。

3.1 常见约束

关键字

  • 非空约束：not null

    限制该字段的数据不能为 null，用户添加数据时没有指定值则会报错

  • 唯一约束：unique

    保证该字段的所有数据都是唯一且不重复的

  • 默认约束：default

    用于指定某列的默认值

  • 检查约束：check(...)

    用于保证字段值满足某一个条件

  • 零填充约束：zerofill

    插入数据时，当该字段的值的长度小于定义的长度时，会在该值的前面补上相应的0

注意

  • zerofill 默认为 int(10)
  • 当使用 zerofill 时，默认会自动加 unsigned 属性，数值范围是原值的2倍

3.2 主键约束

关键字

  • 主键约束：primary key

    主键是一行数据的唯一标识，其数据要求非空且唯一

注意

  • 每个表最多只允许有一个主键
  • 当创建主键约束时，系统默认会在所在的列和列组合上建立对应的唯一索引
  • 对于多列主键，各列数据不能全部重复（唯一性），允许部分重复

3.3 自增长约束

关键字

  • 自增长约束：auto_increment

    当主键定义为自增长后，这个主键的值就不再需要用户输入数据了，而由数据库系统自动赋值，实现主键自增长

注意

  • 一个表中只能有一个字段使用 auto_increment 约束，且该字段必须有唯一索引，以避免序号重复（即为主键或主键的一部分）
  • auto_increment 约束的字段必须具备 not null 属性。
  • auto_increment 约束的字段只能是整数类型（tinyint、smallint、int、bigint 等）。
  • auto_increment 约束字段的最大值受该字段的数据类型约束，如果达到上限，auto_increment 就会失效。
  • 初始值设定规则：
    • 未设置初始值的情况下，auto_increment 的初始值是1，每新增一条记录，字段值自动加1
    • 如果第一条数值设置了 auto_increment 约束的字段的值，则该值作为初始值
  • 删除数据后规则：
    • delete 数据之后，不影响自动增长的计数，仍然在原基础上继续加1
    • truncate 数据之后，无论先前设定的初始值是多少，自动增长重新从默认初始值1开始
  • auto_increment 的实现在 unique 判定之前，当 unique 约束的字段出现重复，数据会添加失败，但 auto_increment 的字段会加1

3.4 外键约束

3.4.1 创建和删除

关键字

  • 关键字：foreign key 字段列表 references 主表名 主键字段列表

    外键用来建立主表与从表的关联关系，为两个表的数据建立连接，约束两个表中数据的一致性和完整性

注意

  • 主表必须已经存在于数据库中，或者是当前正在创建的表。
  • 必须为主表定义主键。
  • 主键数据不能包含空值，但允许在外键数据中出现空值。只要外键的每个非空值出现在指定的主键中，这个外键的内容就是正确的。
  • 字段列表要求：
    • 在主表的表名后面指定列名或列名的组合。这个列或列的组合必须是主表的主键或候选键。
    • 外键中列的数目必须和主表的主键中列的数目相同。
    • 外键中列的数据类型必须和主表主键中对应列的数据类型相同。
  • 删除数据要求：
    • 主表的数据被从表依赖时，不能删除；否则可以删除
    • 从表的数据可以随便删除

3.4.1 删除更新行为

关键字

行为 说明
no action 当在父表中删除/更新对应记录时，首先检查该记录是否有对应外键，如果有则不允许删除/更新
restrict 当在父表中删除/更新对应记录时，首先检查该记录是否有对应外键，如果有则不允许删除/更新
cascade 当在父表中删除/更新对应记录时，首先检查该记录是否有对应外键，如果有,则也删除/更新外键在子表中的记录
set null 当在父表中删除对应记录时，首先检查该记录是否有对应外键，如果有则设置子表中该外键值为 null（要求该外键允许取 null）
set default 父表有变更时，子表将外键列设置成一个默认的值（lnnodb 不支持）

语法

  • 指定外键删除更新行为

    foreign key 字段列表 references 主表名 主键字段列表 on update 更新行为 on delete 删除行为
