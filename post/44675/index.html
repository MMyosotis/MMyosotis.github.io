集合 (二) - Java SE

1. Map 集合

1.1 Map 体系

类的概述

Map 集合是一种双列集合，也叫键值对集合。

每个元素包含两个数据，每个元素的格式：key = value (键值对元素)。

Map 集合的特点有：

  • 集合的特点都是由键决定的。
  • 集合的键无序（存取顺序不一致，但每次的无序是固定的顺序，即取出的顺序虽然与存储顺序不同，但并非每次随机，而是有固定顺序）、不重复、无索引。值不做要求（可以重复）。
  • 集合后面重复的键对应的值会覆盖前面重复键的值。
  • 集合的键值对都可以为 null。

Map 集合常用的实现类有：

  • HashMap：元素按照键是无序，不重复，无索引，值不做要求。（与 Map 体系一致）
  • LinkedHashMap：元素按照键是有序，不重复，无索引，值不做要求。
  • TreeMap：元素按照键是排序，不重复，无索引的，值不做要求

实现类特点

Map 集合的实现类各自存在特点：

实现类 集合特点 底层原理
HashMap 集合 元素按照键无序，不重复，无索引；值不做要求。增删改查快。 哈希表
LinkedList 集合 元素按照键有序，不重复，无索引；值不做要求。增删改查快。 哈希表和双链表
HashSet 集合 元素按照键排序，不重复，无索引；值不做要求。对对象排序。 红黑树

类创建对象

创建 Map 对象只能用其实现类去创建，然后基于多态用父类类型 Map 接收。

1.2 Map 方法

类的常规方法

Map 是双列集合的父接口，它的功能是全部双列集合都可以继承使用的：

方法名称 方法说明
V put(K key, V value) 添加元素
V get(Object key) 根据键获取对应的值
V remove(Object key) 根据键删除键值对元素
void clear() 移除所有的键值对元素
boolean containsKey(Object key) 判断集合是否包含指定的键
boolean containsValue(Object value) 判断集合是否包含指定的值
boolean isEmpty() 判断集合是否为空
int size() 集合的长度，也就是集合中键值对的个数
void putAll(Map<? extends K, ? extends V> m) 将其他Map集合的元素添加至该Map集合
**类的提取和转换方法**

除去常规方法外，Map 中还含有单独提取键或值的集合，以及整体提取键值并转换为 Set 集合的方法：

方法名称 方法说明
Set<K> keySet() 获取全部键的集合
Collection<V> values() 获取全部值的集合
Set<Map.Entry<K, V>> entrySet() 获取所有键值对对象的集合

entrySet() 方法中将 Map 集合转换成了 Set 集合，其中每一个元素为原 Map 集合中的一个键值对，类型为内部接口 Entry 的实现类对象 Map.Entry<K, V>。

1.3 Map 遍历

1.3.1 键找值

先获取 Map 集合的全部键的 Set 集合，遍历键的 Set 集合，然后通过键提取对应值。

1.3.2 键值对

先使用 entrySet() 方法把 Map 集合转换成 Set 集合，Set 集合中每个元素都是键值对实体类型了。遍历 Set 集合，然后提取键以及提取值。

遍历中使用到的方法如下：

方法名称 方法说明
K getKey() 获得 Entry 对象中的键
V getValue() 获取 Entry 对象中的值

1.3.3 lambda表达式

Map 重写了 Iterable 接口中的 forEach 方法，JDK8 推出的 lambda 表达式，提供了一种结合了forEach 方法的更简单、更直接的遍历集合的方式：

方法名称 方法说明
default void forEach(BiConsumer action) 结合lambda遍历集合

1.4 Map 去重

Map 集合是根据键去重的，键去重的规则和 Set 去重是相同的规则。

对于自定义类来说，如果希望 Map 集合认为两个内容一样的对象是重复的键，必须重写类的 hashCode() 和 equals() 方法。equals 方法重写详见前文。

2. Map 实现类

2.1 HashMap 集合

HashMap 是 Map 里面的一个实现类。特点都是由键决定的：无序、不重复、无索引。

HashMap 跟 HashSet 底层原理是一模一样的，都是哈希表结构，增删改查性能都较好，只是 HashMap 的每个元素包含两个值而已。

2.2 LinkedHashMap 集合

LinkedHashMap 的特点是由键决定：有序、不重复、无索引。这里的有序指的是保证存储和取出的元素顺序一致

LinkedHashMap 底层数据结构是依然哈希表，只是每个键值对元素又额外的多了一个双链表的机制记录存储的顺序。

2.3 TreeMap 集合

2.3.1 TreeMap 底层

TreeMap 的特点是由键决定的：不重复、无索引、可排序。可排序是指按照元素的大小默认升序（有小到大）排序。

TreeMap 集合底层是基于红黑树的数据结构实现排序的，增删改查性能都较好。（注意：TreeMap 集合是一定要排序的）

2.3.2 TreeMap 排序

TreeMap 集合的排序规则和 TreeSet 排序是相同的规则。对于自定义类型，TreeMap 无法直接排序，需要手动制定排序规则：

  • 方法一：让自定义的类实现 Comparable 接口重写里面的 compareTo(E o) 方法来定制比较规则。

  • 方法二：TreeMap 集合有参数构造器，可以设置 Comparator 接口对应的比较器对象，来定制比较规则。

3. 集合的嵌套

允许集合中存入的元素仍为一个集合，这即是集合的嵌套。

4. 不可变集合

不可变集合，就是不可被修改的集合。集合的数据项在创建的时候提供，并且在整个生命周期中都不可改变，否则报错。

在 List、Set、Map 接口中，都存在 of(E…elements) 方法，可以创建一个不可变的集合。这个集合不能添加，不能删除，不能修改，但可获取元素：

方法名称 方法说明
static <E> List<E> of(E…elements) 创建一个具有指定元素的List集合对象
static <E> Set<E> of(E…elements) 创建一个具有指定元素的Set集合对象
static <K , V> Map<K，V> of(E…elements) 创建一个具有指定元素的Map集合对象

5. Stream 流

5.1 获取流方法

流的概念

Stream 流结合了 lambda 表达式，用于简化集合、数组操作的 API。

Stream 流的使用方式是，先得到集合或者数组的 Stream 流（就是一根传送带），然后把元素放上去，然后就用这个 Stream 流简化的 API 来方便的操作元素。所有的流对象只能使用一次。

流的获取

  • 集合获取流：使用 Collection 接口中的默认方法 Stream 生成流。Map 集合可以获取到键、值、键值对的集合后分别再调用各自对应流：

    方法名称 方法说明
    default Stream<E> stream() 获取当前集合对象的Stream流

  • 数组获取流：调用 Arrays 中的 Stream 方法，或者 Stream 中的 of(T...values) 方法：

    方法名称 方法说明
    public static <T> Stream<T> stream(T[] array) 获取当前数组的Stream流
    public static<T> Stream<T> of(T... values) 获取当前数组/可变数据的Stream流

5.2 流中间方法

中间方法的概述

中间方法也称为非终结方法，调用完成后返回新的 Stream 流可以继续使用，支持链式编程，但所有的流对象只能使用一次。在 Stream 流中无法直接修改集合、数组中的数据。常用的方法如下：

方法名称 方法说明
Stream<T> filter(Predicate⁢? super T> predicate) 用于对流中的数据进行过滤
Stream<T> limit(long maxSize) 获取前几个元素
Stream<T> skip(long n) 跳过前几个元素
Stream<T> distinct() 去除流中重复的元素,依赖 hashCode 和 equals 方法
static <T> Stream<T> concat(Stream a, Stream b) 合并a和b两个流为一个流

中间加工方法

除此之外还有一个 map 加工方法，用于加工流中元素：

方法名称 方法说明
R> Stream<R> map(Function⁢? super T, ? extends R> mapper) 将流中元素<T>进行加工，并返回加工后的流<R

5.3 流终结方法

一个 Stream 流只能有一个终结方法，是流水线上的最后一个操作，后续无法再有其他操作：

方法名称 方法说明
void forEach(Consumer action) 对此流的每个元素执行遍历操作
long count() 返回此流中的元素数

5.4 收集流

收集流的方法

Stream 中提供了将流收集到集合或数组中的方法 collect ，形参为 Collectors 中的方法。

方法名称 方法说明

Object[] toArray() 将 Stream 流收集到数组
