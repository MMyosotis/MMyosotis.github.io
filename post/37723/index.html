面向对象 (三) - Java SE

1. 内部类

1.1 匿名内部类

匿名内部类将继承、方法重写、创建对象合并为一步，相当于创建了一个子类的对象的同时子类中的方法还进行了重写。

  • 匿名内部类并非真的没有类名，系统会有一个默认的类名并真实创建一个类，但是在编辑器中不会显示。
  • 在含有匿名内部类的局域结束后要加上分号。

1.2 lambda 表达式

语法

lambda 表达式可用于简化函数式接口 (只有一个抽象方的接口，可用 @Functionalinterface 检验) 的匿名内部类的写法。

简写

  • 参数类型可以省略不写。
  • 如果只有一个参数，参数类型可以省略，同时 ( ) 也可以省略。
  • 如果 lambda 表达式的方法体只有一行，则大括号、分号、return 可以省略不写，但必须同时省略。

1.3 方法引用

方法引用是用来直接访问类或者实例的已经存在的方法或者构造方法。

当 lambda 表达式中执行代码块是本身已经存在的方法，则可用方法引用进一步简化。

1.3.1 静态方法引用

静态方法引用，即引用类中的静态方法。lambda 表达式被类方法替代的时候，它的形式参数全部传递给静态方法作为参数。

  • 静态方法引用格式为：类名::静态方法

  • 对应的 lambda 表达式为：(args) -> 类名.静态方法(args)

1.3.2 对象方法引用

对象方法引用，即引用对象的实例方法，需要先创建对象。lambda 表达式被对象的实例方法替代的时候，它的形式参数全部传递给该方法作为参数。

  • 对象方法引用格式为：对象::成员方法

  • 对应的 lambda 表达式为：(args) -> 对象.成员方法(args)

1.3.3 实例方法引用

实例方法引用，即引用类的实例方法。lambda 表达式被类的实例方法替代的时候，第一个参数作为调用者，后面的参数全部传递给该方法作为参数。故而接口方法的形参，第一个参数必须是调用者的类型。

  • 静态方法引用格式为：类名::成员方法

  • 对应的 lambda 表达式为：(对象，args) -> 对象.成员方法(args)

1.3.4 构造方法引用

构造方法引用，即引用类的构造方法。lambda 表达式被构造器替代的时候，它的形式参数全部传递给构造器作为参数。

  • 静态方法引用格式为：类名::new
  • 对应的 lambda 表达式为：(args) -> new 类名(args)

2. 包装类

概念

包装类：即基本数据类型对应的引用类型，包装类和基本类型在 JDK5 之后可以自动装箱、自动拆箱进行转换：

基本类型 byte char int short long float double boolean
包装类 Byte Character Integer Short Long Float Double Boolean

方法

以 Integer 为例，其包含有如下成员方法：

方法名称 方法说明
public static String toBinaryString(int i) 得到二进制
public static String toOctalString(int i) 得到八进制
public static String toHexString(int i) 得到十六进制
public static int parseInt(String s) 将 String 类型的整数转成 int 类型的整数

在八个包装类中除了 Character 以外都有对应的 parseXxx 的方法进行类型转换。

3. 枚举类

概念

枚举是 Java 中的一种特殊类型，枚举的作用是为了做信息的标志和分类。

特点

  • 枚举类都是继承了枚举类型：java.lang.Enum
  • 枚举都是最终类，不可以被继承。
  • 构造器都是私有的，枚举对外不能创建对象。
  • 枚举类的第一行默认都是罗列枚举对象的名称的。
  • 枚举类相当于是多例模式，其中固定创建了多个实例对象。
