面向对象 (二)

1. 继承

1.1 继承的语法

Java 允许我们用 extends 关键字，让一个类和另一个类建立起一种父子关系。子类可以得到父类的属性和行为，子类可以使用。

1.2 继承的特点

继承的特点

  • Java 只支持单继承，一个类只能继承一个直接父类。不支持多继承，但支持多层继承。
  • 每一个类都直接或间接的继承于 Object 类。

子类从父类中继承的内容

父类成员 子类能否继承 子类能否继承
构造方法 非private × private ×
成员变量 非pirvate √ private √
成员方法 虚方法表 √ 非虚方法表 ×

虚方法表：非 private 、static 、 final 的方法。

1.3 super 关键字

super 的使用

super 关键字是一个指向父类的引用，可用于访问父类成员和父类构造方法。

super 和 this 的对比

this ：代表当前方法调用者的地址值。

super ：代表父类存储空间。

关键字 访问成员变量 访问成员方法 访问构造方法
this 访问本类 this.成员变量 访问本类 this.成员方法() 访问本类 this()
访问本类 访问父类 super.成员变量 访问父类 super.成员方法() 访问父类 super()

1.4 子类访问成员

1.4.1 就近原则

在子类方法中访问成员（成员变量、成员方法）满足就近原则：

  • 先子类局部范围找
  • 然后子类成员范围找
  • 然后父类成员范围找，如果父类范围还没有找到则报错。

对于重名变量或重写方法优先就近调用，若要调用远端则要用到 this 和 super 关键字。

1.4.2 访问父类私有成员

父类私有成员能被子类继承下来，但无法被子类直接访问，只能通过父类提供的方法去访问。

1.5 子类方法重写

方法重写的概念

在继承体系中，子类出现了和父类中一模一样的方法声明，我们就称子类这个方法是重写的方法。

在重写方法前加入 @Override 注解可用于检验重写时语法是否正确。

方法重写的注意事项

  • 重写方法的名称、形参列表和返回类型必须与被重写方法一致：

    同名类型 方法规则与父类相同 形参不同 返回类型不同
    继承前后存在同名方法 方法重写 方法重载 冲突，无法实现

  • 只有被添加到虚方法表中的方法才能被重写。

  • 子类重写父类方法时，访问权限必须大于或者等于父类 （缺省 < protected < public ），返回值类型子类必须小于等于父类。

  • 方法重写后，原父类方法仍可访问，访问规则参照就近原则。

1.6 子类构造方法

子类不能继承父类的构造方法，但是可以通过 super 调用。

若子类构造方法未调用父类构造方法，则子类构造方法的第一行有一个默认的 super() ，若手动书写则必须写在第一行。

2. Static 关键字

2.1 静态变量

定义静态变量

static 修饰后的成员变量，可以被类的所有对象共享，访问，修改。因为类只有一个，所以静态变量在内存区域中随类的加载进入堆中的静态区并唯一存在。

访问静态变量

可以采用类名和对象名去访问静态成员变量。

2.2 静态方法

定义静态方法

有 static 修饰成员方法称为静态方法，所有对象都可以共享这个方法。因为类只有一个，所以静态方法在内存区域中也只存在一份。

静态方法多用于测试类与工具类中。

访问静态方法

可以采用类名和对象名去访问静态成员变量。

静态方法的访问规则

  • 静态方法只能访问静态变量和静态方法，不可以直接访问实例成员。
  • 实例方法可以访问实例成员，也可以访问静态成员，但无法使用 super 和 this 关键字去访问静态成员。
  • 静态方法中没有 this 关键字。（所以才有第一条规则，因为实例方法默认有指向调用者地址值的 this 作形参。）

2.3 静态代码块

静态代码块和实例代码块的区别如下：

  • 静态代码块：是通过 static 关键字修饰的代码块，随着类的加载而加载，并且自动触发、只执行一次。
  • 实例代码块：无 static 修饰，属于对象，每次构建对象时，都会触发一次执行。

2.4 main 方法

在 Java 中，main 方法是 Java 应用程序的入口方法，程序在运行的时候，第一个执行的方法就是 main 方法，且是静态方法不必创建对象，随着类被加载进内存时该方法即可调用。

3. final 关键字

final 关键字是最终的意思，可以修饰类、方法、变量：

  • 修饰类：表明该类是最终类，不能被继承。

  • 修饰方法：表明该方法是最终方法，不能被重写。

  • 修饰变量：即常量，表示该变量第一次赋值后，不能再次被赋值，有且仅能被赋值一次。

    • final 修饰的变量是基本类型：那么变量存储的数据值不能发生改变。

    • final 修饰的变量是引用类型：那么变量存储的地址值不能发生改变，但是地址指向的对象内容是可以发生变化的。

4. 多态

4.1 多态的实现

多态是指对象可以有多种形态，多态的前提是：有继承关系、有父类引用指向子类对象、有方法重写。

语法格式为：父类类型 对象名称 = new 子类对象。

4.2 调用成员

采用父类引用接收了子类对象的地址后，对象调用成员有一定规则：

  • 方法调用：编译时找父类中该方法是否存在，运行时调用子类中该方法。所以多态不能调用子类的特有方法。
  • 变量调用：编译时找父类中该变量是否存在，运行时调用父类中该变量。所以多态不能调用子类的特有成员。

4.3 类型转换

  • 从子类到父类：为自动进行的类型转换。

  • 从父类到子类：类型转换需要强制进行，且若原父类类型为子类的多态用法，则转换成子类时只能转换对应子类。

    ​ 为了避免强制转换子类的类型出错，常用 instanceof 判断其真实类型，再进行转换。

5. 抽象类和方法

抽象类和方法的概念

关键字 abstract 可以修饰类、成员方法。abstract 修饰类时这个类就是抽象类；修饰方法时这个方法就是抽象方法。

抽象类为子类规定了方法重写格式，抽象方法只有方法签名，没有方法体。

抽象类的特点

  • 抽象类不能实例化，类有的成员（成员变量、方法、构造器）抽象类都具备。
  • 抽象类的子类：
    • 子类重写抽象类中所有抽象方法，成为非抽象类。
    • 子类未重写所有抽象方法，仍为抽象类。
  • 抽象类中不一定有抽象方法，有抽象方法的类一定是抽象类。

6. 接口

6.1 接口的定义

接口的概念

接口用关键字 interface 定义，用于规定某一类行为的规则，是对行为的抽象。其书写格式不含形参列表。

接口的成员

  • 接口的对象：接口不能实例化，所以没有对象，若想访问成员只能通过接口名。
  • 接口中的成员变量：只能是静态常量，默认修饰符 public static final。
  • 接口中的构造方法：没有构造方法。
  • 接口中的成员方法：JDK7以前只能是抽象方法，默认修饰符 public abstract。

6.2 接口的实现

使用关键字 implements 可以使接口用来被类实现的，实现接口的类称为实现类。

6.3 接口的特点

接口与类之间的关系

  • 一个类可以实现多个接口，这区别于类只能单继承。并且类可以在继承父类的同时实现接口：

    同名类型 方法规则相同 形参不同 返回类型不同
    父类与接口存在同名方法 无需重写接口方法
    父类继承的方法视作接口的实现    		 需重写接口方法
     同名的两者视为方法重载     		 需重写接口方法
     同名的两者视为方法重载 
    接口与接口存在同名方法 只需重写一次接口方法 两者接口方法均需重写
    同名的两者视为方法重载    		 规则冲突，不可实现

  • 接口的子类：

    • 实现类重写接口中所有抽象方法，成为非抽象类。
    • 实现类未重写接口中所有抽象方法，为抽象类。

接口与接口之间的关系

  • 一个接口可以继承多个接口，可以将多个接口整合为同一个，但继承来的方法规则不能冲突：

    同名类型 方法规则相同 形参不同 返回类型不同
    继承前后存在同名方法 视为接口中一个方法 视为接口中不同方法(重载) 冲突，不可继承
