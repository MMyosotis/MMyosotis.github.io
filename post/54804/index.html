IO 流 (二) - Java SE

1. 缓冲流

缓冲流也称为高效流、或者高级流。缓冲流自带在流和内存之间的缓冲区、可以提高原始字节流、字符流读写数据的性能。缓冲区提高性能的原理是通过增加每次交互的数据总量来减少磁盘交互次数和系统调用。

缓冲流主要有四个实现类：

  • 缓冲输入流：自带了8KB缓冲池，以后直接从缓冲池读取数据，所以性能较好。
  • 缓冲输出流：自带了8KB缓冲池，数据就直接写入到缓冲池中去，写数据性能提高了。

1.1 字节缓冲流

1.1.1 输入与输出

概述

字节缓冲流分为输入流和输出流两大类：

  • 字节缓冲输入流：BufferedInputStream 。

  • 字节缓冲输出流：BufferedOutputStream 。

    ​ 想要实现追加而不是覆盖，则被包装的 FileOutputStream 应采用追加数据的构造方法。

BufferedInputStream 和 BufferedOutputStream 的构造方法如下：

方法名称 方法说明
public BufferedInputStream(InputStream is) 可以把低级的字节输入流包装成一个高级的缓冲字节输入流管道
public BufferedOutputStream(OutputStream os) 可以把低级的字节输出流包装成一个高级的缓冲字节输出流

方法

BufferedInputStream 和 BufferedOutputStream 也分别从 InputStream 和 OutputStream 继承了基本的读取和输出数据的API，以及刷新和关闭流的 API，方法使用和注意事项同基本字节流，但缓冲字节流性能更优。

同时创建一个字节输入流和字节输出流，由输入流读取源文件数据，再将其输出到新文件，代码详见资源释放。

总体来说，使用字节数组进行读取输出速度要快于单个字节，使用缓冲流速度要快于基本流，如下是一个速度测试案例。

1.2 字符缓冲流

1.2.1 输入与输出

概述

  • 字符缓冲输入流：BufferedReader 。

  • 字符缓冲输出流：BufferedWriter 。

    ​ 想要实现追加而不是覆盖，则被包装的 FileWriter 应采用追加数据的构造方法。

BufferedReader 和 BufferedWriter 的构造方法如下：

方法名称 方法说明
public BufferedReader(Reader r) 可以把低级的字符输入流包装成一个高级的缓冲字符输入流管道
public BufferedWriter(Writer w) 可以把低级的字符输出流包装成一个高级的缓冲字符输出流管道

方法

BufferedReader 和 BufferedWriter 也分别从 Reader 和 Writer 继承了基本的读取和输出数据的API，以及刷新和关闭流的 API，方法使用和注意事项同基本字符流，但缓冲字节流性能更优。

1.2.2 新增方法

输入流的新增方法

BufferedReader 新增了读取一行的功能：

方法名称 方法说明
public String readLine() 读取一行数据返回，如果读取没有完毕，无行可读返回null

输出流的新增方法

BufferedWriter 新增了换行操作：

方法名称 方法说明
public void newLine() 换行操作

2. 转换流

在读取和输出字符时，必须文件和代码编码一致才不会乱码。为了解决编码格式不一致，可以使用转换流。

英文和数字的编码在各字符集中通用，所以不会出现乱码。

2.1 字符输入转换流

按照指定编码格式读取文件共有使用转换流和不使用转换流两种方式。

方式一：使用字符输入转换流

使用字符输入转换流 InputStreamReader，可以把字节输入流按照指定编码转换成字符输入流，然后再输入。

InputStreamReader 的构造方法如下：

方法名称 方法说明
public InputStreamReader(InputStream is) 可以把原始的字节流按照代码默认编码（UTF-8）转换成字符输入流，与默认的 FileReader 一样
public InputStreamReader(InputStream is ，String charset) 可以把原始的字节流按照指定编码转换成字符输入流

除此之外 InputStreamReader 从 Reader 继承了基本的读取数据的API，方法使用和注意事项同基本字符输入流。

方式二：使用字节流读取并解码

使用字节输入流 FileInputStream 读取文件字节后，按照指定编码格式使用 String 构造器进行解码，解码方式详见字符集的编码与解码。

2.2 字符输出转换流

按照指定编码格式输出文件共有使用转换流和不使用转换流两种方式。

方法一：使用字符输出转换流

使用字符输出转换流 OutputStreamWriter，可以把字节输出流按照指定编码转换成字符输出流，然后再输出。

OutputStreamWriter 的构造方法如下：

方法名称 方法说明
public OutputStreamWriter(OutputStream os) 可可以把原始的字节输出流按照代码默认编码（UTF-8）转换成字符输出流，与默认的 FileWriter 一样
public OutputStreamWriter(OutputStream os，String charset) 可以把原始的字节输出流按照指定编码转换成字符输出流

除此之外 OutputStreamWriter 从 Writer 继承了基本的输出数据和刷新、关闭流的API，方法使用和注意事项同基本字符输出流。

方式二：编码后使用字节流输出

使用 byte[] getBytes(String chName) 方法把字符以指定编码获取字节后，再使用字节输出流 FileOutputStream 写出去，编码方式详见字符集的编码与解码。

3. 序列化

对象的序列化即把内存中的对象存储到磁盘文件中去，反序列化即把磁盘文件中的对象读取到内存中来。

3.1 对象序列化

概述

把内存中的对象存储到磁盘文件中去，称为对象序列化，实现类是对象字节输出流 ObjectOutputStream 。

ObjectOutputStream 的构造方法如下：

方法名称 方法说明
public ObjectOutputStream(OutputStream out) 把低级字节输出流包装成高级的对象字节输出流
**方法**

除了从 OutputStream 继承了基本的输出数据以及刷新和关闭流的 API 外，ObjectOutputStream 提供了输出对象的方法：

方法名称 方法说明
public final void writeObject(Object obj) 把对象写出去到对象序列化流的文件中去
在将对象序列化时有几个注意事项：
  • 如果对象要序列化，则类必须实现 Serializable 序列化接口。
  • 对象的属性中，使用了 transient 关键字的成员将不被序列化。

3.2 对象反序列化

概述

把存储到磁盘文件中去的对象数据恢复成内存中的对象，称为对象反序列化，实现类是对象字节输入流 ObjectInputStream 。

ObjectInputStream 的构造方法如下：

方法名称 方法说明
public ObjectInputStream(InputStream out) 把低级字节输如流包装成高级的对象字节输入流
**方法**

除了从 InputStream 继承了基本的输出数据以及刷新和关闭流的 API 外，ObjectInputStream 提供了读取对象的方法：

方法名称 方法说明
public Object readObject() 把存储到磁盘文件中去的对象数据恢复成内存中的对象返回

在将对象序反列化时有几个注意事项：

  • 由于对象的属性中，使用了 transient 关键字的成员将不被序列化，在反序列化时该属性将执行默认初始化。
  • 若在类中申明了序列化的版本号 serialVersionUID（修饰符为 public static final，类型为 long）则序列化时类的版本号和反序列化时类的版本号必须一致，否则报错。

4. 打印流

打印流可以实现方便、高效的打印数据到文件中去。调用其打印方法时，指定的数据内容是什么就会原封不动打印到文件中。

共包括 PrintStream、PrintWriter 两个实现类：二者打印功能并无差别。

  • PrintStream ：继承自字节输出流 OutputStream，除了打印外还支持输出字节数据的方法。
  • PrintWriter：继承自字符输出流 Writer，除了打印外还支持输出字符数据的方法。

4.1 打印的方法

概述

字节打印输出流 PrintStream 的构造方法如下：想要实现追加而不是覆盖，则被包装的 FileOutputStream 应采用追加数据的构造方法。

方法名称 方法说明
public PrintStream(OutputStream os) 打印流直接通向字节输出流管道
public PrintStream(File f) 打印流直接通向文件对象
public PrintStream(String filepath) 打印流直接通向文件路径
字符打印输出流 `PrintWriter` 的构造方法如下：想要实现追加而不是覆盖，则被包装的 `FileWriter` 应采用[追加数据的构造方法](/post/5808/#4-2-2-字符输出流)。
方法名称 方法说明
public PrintWriter(OutputStream os) 打印流直接通向字节输出流管道
public PrintWriter (Writer w) 打印流直接通向字符输出流管道
public PrintWriter (File f) 打印流直接通向文件对象
public PrintWriter (String filepath) 打印流直接通向文件路径
**方法**

除了分别从 OutputStream 和 Writer 继承了基本的输出数据以及刷新和关闭流的 API 外，PrintStream 和 PrintWriter 都提供了打印的方法：

方法名称 方法说明
public void print(E e) 打印任意类型的数据出去，不换行
public void println(E e) 打印任意类型的数据出去并换行
public void printf(String format, Object... args) 打印字符串，允许使用格式字符

三种打印方式的具体使用和在控制台输出时的效果是一致的。

4.2 输出重定向

System.out 实际返回的是一个 PrintStream 默认指向控制台的打印流。

可以通过 System.setOut(PrintStream ps) 重新指定打印的路径。

5. 属性集

概述

Properties 属于 Map 集合体系，继承于 Hashtable，来表示一个持久的属性集。它可以把自己对象中的键值对信息存入到一个属性文件（文件后缀 .properties）中去，每个键及其对应值都是一个字符串。除此之外他它也具备集合本身的性质。

Properties 构造方法一般使用其默认无参构造方法。

方法

属性集新增了与 IO 流结合的读取和输出方法：

方法名称 方法说明
void load(InputStream inStream) 从输入字节流读取属性列表（键和元素对）
void load(Reader reader) 从输入字符流读取属性列表（键和元素对）
void store(OutputStream out, String comments) 使用字符流将此属性列表（键和元素对）写入此 Properties 表中
void store(Writer writer, String comments) 使用字符流将此属性列表（键和元素对）写入此 Properties 表中
public Object setProperty(String key, String value) 保存键值对 (put)
public String getProperty(String key) 使用此属性列表中指定的键搜索属性值 (get)
public Set<String> stringPropertyNames() 所有键的名称的集合 (keySet())

最后三个新增方法实际上等同于 Map 的方法，但是 Properties 的方法规定了键对值的类型只能是 String。

使用字符流和字节流的区别如下：

  • 使用字节流输出，属性文件中汉字会出现乱码。读取时使用字节流或者字符流读入均正常。
  • 使用字符流输出，属性文件中汉字不会出现乱码。读取时只能使用字符流读入，使用字节流读入则会显示乱码。

6. IO 框架

lcommons-io 是 apache 开源基金组织提供的一组有关 IO 操作的类库，可以提高 IO 功能开发的效率。

lcommons-io 工具包提供了很多有关 IO 操作的类。有两个主要的类 FileUtils，IOUtils。

FileUtils 主要有如下方法：

方法名称 方法说明
String readFileToString(File file, String encoding) 读取文件中的数据, 返回字符串
void copyFile(File srcFile, File destFile) 复制文件
void copyDirectoryToDirectory(File srcDir, File destDir) 复制文件夹
