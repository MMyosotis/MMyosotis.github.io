IO 流 (一)

1. File 类

1.1 创建对象

类的概述

File 类在 java.io.File 包下，代表操作系统文件（文件、文件夹）的对象。

File 对象可以定位文件和文件夹。File 封装的对象仅仅是一个路径名，这个路径可以是存在的，也可以是不存在的。

方法名称 方法说明
public File(String pathname) 根据文件路径创建文件对象
public File(String parent, String child) 根据父路径名字符串和子路径名字符串创建对象
public File(File parent, String child) 根据父路径对应文件对象和子路径名字符串创建对象

绝对和相对路径

  • 绝对路径：从盘符开始的完整路径。

  • 相对路径：相对指的是相对于当前项目的根目录的路径。

1.2.3 遍历文件夹

File 类提供了获取当前目录下所有一级文件的方法：

方法名称 方法说明
public String[] list() 获取当前目录下所有的"一级文件名称(带后缀)"到一个字符串数组中去返回
public File[] listFiles() （常用） 获取当前目录下所有的"一级文件对象"到一个文件对象数组中去返回
`listFiles` 方法注意事项：
  • 当调用对象不存在时或者代表文件时，返回 null。
  • 当调用对象代表一个空文件夹时，返回一个长度为0的数组。
  • 当调用对象是一个有内容的文件夹时，将里面所有文件和文件夹的路径放在 File 数组中返回。
  • 当调用对象是一个有隐藏文件的文件夹时，将里面所有文件和文件夹的路径放在 File 数组中返回，包含隐藏文件。
  • 当没有权限访问该调用对象文件夹时，返回 null。

2. 字符集

2.1 字符集概述

计算机可以给字符进行编号存储，这套编号规则就是字符集。

字符集是多个字符的集合，字符集种类较多，每个字符集包含的字符个数不同，常见字符集有：

  • ASCII 字符集：
    • ASCII (美国信息交换标准代码)：包括了数字、英文、符号。
    • ASCII使用一个字节存储一个字符，一个字节是8位，总共可以表示128个字符信息。
  • GBK 字符集：
    • GBK是中国的码表，包含了几万个汉字等字符，支持繁体字和部分日韩文字，同时也要兼容ASCII编码。
    • GBK编码中一个中文字符一般以两个字节的形式存储。
  • Unicode 字符集：
    • 统一码，也叫万国码，容纳世界上大多数国家的所有常见文字和符号。
    • Unicode会先通过UTF-8，UTF-16，UTF-32编码成二进制后再存储到计算机。其中最为常见的就是UTF-8，UTF-8编码后一个中文一般以三个字节的形式存储，同时也要兼容ASCII编码表。
    • 是计算机科学领域里的一项业界标准。技术人员都应该使用 UTF-8 的字符集编码。

英文和数字在任何国家的字符集中都占一个字节。

3. IO 流体系

IO 流的概念

IO 流包括了输入和输出：

  • I 表示 intput，把硬盘文件中的数据读入到内存的过程，称之输入，负责读。
  • O 表示 output，把内存中的数据写出到硬盘文件的过程，称之输出，负责写。

IO 流的分类

总体可以将 IO 流分为如下几类：

  • 字节流：操作所有类型文件。
    • 字节输入流：以内存为基准，来自磁盘文件/网络中的数据以字节的形式读入到内存中去。
    • 字节输出流：以内存为基准，把内存中的数据以字节写出到磁盘文件或者网络中去。
  • 字符流：只能操作纯文本文件。
    • 字符输入流：以内存为基准，来自磁盘文件/网络中的数据以字符的形式读入到内存中去。
    • 字符输出流：以内存为基准，把内存中的数据以字符写出到磁盘文件或者网络介质中去。

4. 基本流

基本流主要有四个实现类如下图：

  • 字节流：适合做一切文件数据的拷贝（音视频，文本），字节流不适合读取中文内容输出。
  • 字符流：适合做文本文件的操作（读，写）。

4.1 字节流

4.1.1 字节输入流

字节输入流的概述

文件字节输入流的实现类：FileInputStream 。

FileInputStream 构造方法如下：

方法名称 方法说明
public FileInputStream(File file) 创建字节输入流管道与源文件对象接通
public FileInputStream(String pathname) 创建字节输入流管道与源文件路径接通

字节输入流的方法

FileInputStream 继承了 InputStream 中提供的读取数据的方法：

方法名称 方法说明
public int read() 每次读取一个字节返回读取内容，如果字节已经没有可读的返回-1
public int read(byte[] buffer) 每次读取一个字节数组并返回读取字节数，如果字节已经没有可读的返回-1
public int read(byte[] b, int off, int len) 每次读取一个字节数组的一部分并返回读取字节数，如果字节已经没有可读的返回-1

两种读取方法的注意事项如下：

  • 按单个字节读取：在读取由多个字节存储的字符时也只能逐个读取字节，若逐个输出则会出现乱码，只能读取完后存放于一个数组中然后解码。

  • 按字节数组读取：对于多个字节存储的字符出现的问题和处理方式同上。同时若最后一轮读取的字节未填满数组，则数组中未填入新读取字节的位置保留的是上一轮读取的内容而非空。

字节一次性输入

若想要一次性读取完文件内的所有内容有两种方法：

  • 定义一个字节数组与文件大小一样大，一次性读取完成。

  • 使用官方提供的方法：官方为字节输入流 InputStream 提供了如下方法可以吧文件的全部数据读取到一个字节数组中。

    方法名称 方法说明
    public byte[] readAllBytes() throws IOException 直接将当前字节输入流对应的文件对象的字节数据装到一个字节数组返回

字节输出流的概述

文件字节输出流的实现类：FileOutputStream  。

FileOutputStream 构造方法如下：

源文件若不存在，在创建流时会自动创建文件，但路径必须存在。若选择追加数据，则 boolean 处应填入 true 。

方法名称 方法说明
public FileOutputStream(File file) 创建字节输出流管道与源文件对象接通，会清空旧数据
public FileOutputStream(File file，boolean append) 创建字节输出流管道与源文件对象接通，可追加数据
public FileOutputStream(String filepath) 创建字节输出流管道与源文件路径接通，会清空旧数据

字节输出流的方法

FileOutputStream 继承了 OutputStream 中提供的输出数据的 API：

方法名称 方法说明
public void write(int a) 写一个字节出去
public void write(byte[] buffer) 写一个字节数组出去
public void write(byte[] buffer , int pos , int len) 写一个字节数组的一部分出去

FileOutputStream 还继承了 OutputStream 中关于流的刷新与关闭的 API：

输出完数据后必须刷新流，缓存在内存中的数据才能在文件中生效。输出结束后必须关闭流，释放资源。

方法名称 方法说明
flush() 刷新流，将内存中的数据立刻写出，流还可以继续使用
close() 关闭流，释放资源，但是在关闭之前会先刷新流。一旦关闭，就不能再写数据

两种输出方法的注意事项如下：

  • 按单个字节输出：在输出由多个字节存储的字符时也只能输出第一个字节，输出的结果会出现乱码。

  • 按字节数组输出：对于多个字节存储的字符若不完整输出全部字节，也会出现乱码。可以将其转换成字节数组再输出，则不会出现乱码。

4.2 字符流

4.2.1 字符输入流

字符输入流的概述

文件字符输入流的实现类：FileReader 。

FileReader 构造方法如下：

方法名称 方法说明
public FileReader(File file) 创建字符输入流管道与源文件对象接通
public FileReader(String pathname) 创建字符输入流管道与源文件路径接通

字符输入流的方法

FileReader 继承了 Reader 中提供的读取数据的 API：

方法名称 方法说明
public int read() 每次读取一个字符返回读取内容，如果字符已经没有可读的返回-1
public int read(char[] buffer) 每次读取一个字符数组，返回读取的字符个数，如果字符已经没有可读的返回-1
public abstract int read(char[] cbuf, int off, int len) 每次读取一个字符数组的一部分，返回读取的字符个数，如果字符已经没有可读的返回-1

两种读取方法的注意事项如下：

  • 按单个字符读取：相较于字节输入流中每次读取一个字节，字符输入流能完整读取一个中文字符而不会出现乱码。

  • 按字符数组读取：若最后一轮读取的字符未填满数组，则数组中未填入新读取字符的位置保留的是上一轮读取的内容而非空。

源文件若不存在，在创建流时会自动创建文件。若选择追加数据，则 boolean 处应填入 true 。

方法名称 方法说明

close() 关闭流，释放资源，但是在关闭之前会先刷新流。一旦关闭，就不能再写数据

4.3 输出换行

换行符号

输出一个换行符到文件中共有两种写法：

  • 直接输出换行符号：Mac 中为 "\r"，Unix/Linux 中为 "\n"，Windows 中为 "\r\n" 。
  • 输出 Java 自定义的换行符号（通用）：System.getProperty("line.separator") ，即获取当前操作系统的换行符。

字节输出流换行

在字节流中对于换行符均需要调用 getBytes() 方法获取其对应字节，然后再输出。

字符输出流换行

字符流中换行符可以直接输出，不需要转换成对应字节。
